## Development

Run from the project root:

| Command | Description |
|---------|-------------|
| `npm run dev` | Start frontend (Vite) and backend (tsx) concurrently |
| `npm run dev:frontend` | Start frontend dev server only (port 5173) |
| `npm run dev:backend` | Start backend dev server only (port 3100) |
| `npm run build` | Build both frontend and backend for production |
| `npm run build:frontend` | Build frontend only (Vite → `frontend/dist/`) |
| `npm run build:backend` | Build backend only (tsc → `backend/dist/`) |
| `npm run install:all` | Install dependencies in root, frontend, and backend |

## Type Checking

A `type-check` script (added in v1.38.0) type-checks without bundling; the per-project builds also type-check:

| Command | Description |
|---------|-------------|
| `npm run type-check` | Type-checks frontend + backend, no emit (added v1.38.0) |
| `cd frontend && npm run type-check` | Type-checks the frontend (`tsc -b --noEmit`) |
| `cd backend && npm run type-check` | Type-checks the backend (`tsc --noEmit`) |
| `cd frontend && npm run build` | Type-checks the frontend (`tsc -b`) then bundles with Vite |
| `cd backend && npm run build` | Type-checks and compiles the backend (`tsc`) |

## Template Versioning

Used when making changes to the **starter template itself** (not in cloned apps):

| Command | Description |
|---------|-------------|
| `npm run template:version:patch` | Bump patch version (e.g., 1.15.7 → 1.15.8) |
| `npm run template:version:minor` | Bump minor version (e.g., 1.15.7 → 1.16.0) |
| `npm run template:version:major` | Bump major version (e.g., 1.15.7 → 2.0.0) |

<Aside type="caution">
  Always bump the template version after modifying code files. See [Developer Workflow](/starter-template/developer-workflow/) for what triggers a bump.
</Aside>

## Dependency Updates

Run from the project root to update dependencies across root, frontend, and backend (cross-platform; preserves `npm link` symlinks):

| Command | Description |
|---------|-------------|
| `npm run deps:check` | Preview pending updates (read-only, no changes) |
| `npm run deps:update` | Apply safe, semver-respecting updates (`npm update`) |
| `npm run deps:update:latest` | Bump everything to latest via npm-check-updates (may break) |
| `npm run deps:lockfile` | Regenerate lockfiles from `package.json` without installing `node_modules` |
| `npm run deps:relink` | Re-create `npm link` symlinks from the last snapshot |

See [Scripts → Dependency Updates](/starter-template/scripts/#dependency-updates) for the full reference.

## Database (Prisma)

Run from the `backend/` directory:

| Command | Description |
|---------|-------------|
| `npx prisma migrate dev --name describe_change` | Create and apply a new migration |
| `npx prisma migrate deploy` | Apply pending migrations (production) |
| `npx prisma generate` | Regenerate the Prisma client |
| `npx prisma studio` | Open the Prisma database GUI |
| `npx prisma migrate reset` | Reset the database (drops all data) |
| `npx prisma db push` | Push schema to database without migrations |
| `npx prisma validate` | Validate the Prisma schema |
| `npx prisma format` | Format the Prisma schema file |

`backend/package.json` also defines short npm-script aliases for the most common Prisma operations:

| Command | Equivalent |
|---------|------------|
| `npm run db:generate` | `prisma generate` |
| `npm run db:migrate` | `prisma migrate dev` |
| `npm run db:migrate:deploy` | `prisma migrate deploy` |
| `npm run db:studio` | `prisma studio` |

<Aside type="tip">
  The Prisma CLI config is in `backend/prisma.config.ts`. It sets the Prisma schema path, migrations path, and the database URL (`DIRECT_DATABASE_URL` for migrations, falling back to `DATABASE_URL`). The runtime adapter (neon or pg) is configured separately in `backend/src/lib/prisma.ts`.
</Aside>

## shadcn Components

Add UI components from the `frontend/` directory:

```bash
cd frontend
npx shadcn@latest add button
npx shadcn@latest add card
npx shadcn@latest add dialog
npx shadcn@latest add dropdown-menu
npx shadcn@latest add input
npx shadcn@latest add label
npx shadcn@latest add select
npx shadcn@latest add switch
npx shadcn@latest add table
npx shadcn@latest add tabs
npx shadcn@latest add textarea
npx shadcn@latest add tooltip
```

Components are added to `frontend/src/components/ui/`. Do not edit these files directly — they can be overwritten by shadcn updates.

## Git Commands

### Template Updates (Cloned Apps)

```bash
# Add template remote (first time only)
git remote add template-upstream https://github.com/ui-git-kim/unique-react-fullstack-starter.git

# Check for updates
git fetch template-upstream

# Apply updates
git merge template-upstream/main
```

### Documentation Submodule

```bash
# Push docs changes
cd docs && git add . && git commit -m "Docs: description" && git push

# Update submodule reference in main repo
cd .. && git add docs && git commit -m "Update docs submodule"
```

## Troubleshooting

### Clear Caches

```bash
rm -rf node_modules
rm -rf frontend/node_modules
rm -rf backend/node_modules
npm run install:all
```

### Reset Database

```bash
cd backend
npx prisma migrate reset --force
```

### Check Ports

```bash
# macOS/Linux
lsof -i :5173  # Frontend
lsof -i :3100  # Backend

# Windows
netstat -ano | findstr :5173
netstat -ano | findstr :3100
```

### Prisma Client Issues

```bash
cd backend
npx prisma generate
```

If the Prisma client is out of sync after schema changes, regenerate it.
