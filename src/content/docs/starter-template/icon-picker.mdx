The icon picker is a shared component library at `@/components/icon-picker` that all modules must use when they need icon selection. It provides headless primitives for full control and composed components for common use cases.

<Aside type="note">
The headless primitives (`useIconPicker`, `IconPickerGrid`, `IconRenderer`) depend only on React and `lucide-react`. The composed components (`IconPicker`, `IconPickerDialog`) additionally require shadcn `Button`, `Input`, and `Dialog`.
</Aside>

---

## Quick Start

The simplest way to add icon selection is the dialog variant — a button that opens a searchable picker:

```tsx
import { useState } from 'react';
import { IconPickerDialog } from '@/components/icon-picker';

function MyComponent() {
  const [icon, setIcon] = useState<string | null>(null);

  return <IconPickerDialog value={icon} onChange={setIcon} allowNone />;
}
```

For an inline picker without a dialog wrapper:

```tsx
import { IconPicker } from '@/components/icon-picker';

<IconPicker onChange={(icon) => console.log(icon)} allowNone />
```

## Headless Primitives

Use these when you need full control over layout and styling.

### `useIconPicker()`

Hook that loads all Lucide icons, merges search tags, and provides filtered results:

```tsx
const {
  search,        // Current search string
  setSearch,     // Update search
  category,      // Active category filter (null = all)
  setCategory,   // Set category filter
  categories,    // All available categories (readonly array)
  icons,         // Filtered IconData[] based on search + category
} = useIconPicker();
```

Each `IconData` in the `icons` array contains:

| Field | Type | Description |
|-------|------|-------------|
| `name` | `string` | PascalCase component name (e.g. `"ActivitySquare"`) |
| `friendlyName` | `string` | Human-readable name (e.g. `"Activity Square"`) |
| `Component` | `LucideIcon` | The React component — render directly |
| `tags` | `string[]` | Search tags (e.g. `["pulse", "health", "monitor"]`) |

Search matches against the icon name, friendly name, and tags simultaneously.

### `IconRenderer`

Render any Lucide icon by its PascalCase name. Accepts all SVG props:

```tsx
<IconRenderer icon="Activity" className="size-5" />
<IconRenderer icon="UnknownIcon" fallback={<span>?</span>} />
```

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `icon` | `string` | — | PascalCase icon name |
| `fallback` | `ReactNode` | `null` | Shown when the icon name is not found |
| `strokeWidth` | `number` | `1` | SVG stroke width |
| `...props` | `SVGProps` | — | Passed through to the SVG element |

### `IconPickerGrid`

Headless grid that delegates all rendering to callbacks:

```tsx
<IconPickerGrid
  icons={icons}
  onSelect={(name) => setSelected(name)}
  allowNone
  search={search}
  className="grid grid-cols-[repeat(auto-fill,minmax(44px,1fr))] gap-2"
  renderIcon={(icon, onSelect) => (
    <button key={icon.name} onClick={onSelect}>
      <icon.Component className="size-5" />
    </button>
  )}
  renderNone={(onSelect) => (
    <button onClick={onSelect}>None</button>
  )}
  renderEmpty={() => <p>No icons found.</p>}
/>
```

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `icons` | `IconData[]` | — | Filtered icons from `useIconPicker` |
| `onSelect` | `(icon: string \| null) => void` | — | Selection callback (`null` for "none") |
| `renderIcon` | `(icon: IconData, onSelect: () => void) => ReactNode` | — | Render each icon cell |
| `allowNone` | `boolean` | `false` | Show a "none" option |
| `renderNone` | `(onSelect: () => void) => ReactNode` | — | Render the "none" cell |
| `renderEmpty` | `() => ReactNode` | — | Render when no icons match |
| `search` | `string` | `""` | Hides "none" option while searching |
| `className` | `string` | — | Grid container class names |

## Composed Components

Ready-to-use components that combine the headless primitives with shadcn UI.

### `IconPicker`

Search input with a scrollable icon grid:

```tsx
<IconPicker
  onChange={(icon) => setSelected(icon)}
  allowNone
  className="max-w-md"
/>
```

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `onChange` | `(icon: string \| null) => void` | — | Called when an icon is selected |
| `allowNone` | `boolean` | `false` | Show option to clear the selection |
| `className` | `string` | — | Additional class names |

Controlled dialog that wraps `IconPicker`. Shows the current selection on the trigger button:

```tsx
<IconPickerDialog
  value={icon}
  onChange={setIcon}
  allowNone
/>
```

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string \| null` | — | Currently selected icon name |
| `onChange` | `(icon: string \| null) => void` | — | Called when selection changes |
| `allowNone` | `boolean` | `false` | Show option to clear the selection |
| `className` | `string` | — | Additional class names for the trigger button |

## Bundle size and lazy loading

`useIconPicker` loads the **entire** Lucide icon set and merges the `lucide-tags.json` search metadata, then memoises search and category filtering. Enumerating every icon is the whole point of a *browse-everything* picker — but it also means the picker **cannot be tree-shaken**: it deliberately pulls in all of Lucide.

<Aside type="caution" title="Reach the picker only through a dynamic import">
Because it enumerates the full icon set, the picker must live in its **own lazy chunk**. Never import `IconPicker` or `IconPickerDialog` eagerly into a route or a shared bundle — every icon would ship in that bundle. Lucide's own docs warn that importing the whole `icons` object "is not recommended when you are using a bundler since your application build size will grow substantially." Load the picker on demand, so its code (and all of Lucide) is fetched only when a user actually opens it:

```tsx
import { lazy, Suspense, useState } from 'react';

const IconPickerDialog = lazy(() =>
  import('@/components/icon-picker').then((m) => ({ default: m.IconPickerDialog })),
);

function IconField() {
  const [icon, setIcon] = useState<string | null>(null);
  return (
    <Suspense fallback={null}>
      <IconPickerDialog value={icon} onChange={setIcon} allowNone />
    </Suspense>
  );
}
```
</Aside>

## Categories

The `CATEGORIES` constant is a readonly array of all 41 Lucide icon categories. `CATEGORY_LABELS` maps each category to a human-friendly label.

```tsx
import { CATEGORIES, CATEGORY_LABELS } from '@/components/icon-picker';
import type { Category } from '@/components/icon-picker';

// Build a category filter
CATEGORIES.map((cat) => (
  <button key={cat} onClick={() => setCategory(cat)}>
    {CATEGORY_LABELS[cat]}
  </button>
));
```

## Storing Icon Selections

Icons are identified by their PascalCase component name (e.g. `"Activity"`, `"AlertCircle"`). Store this string in your database or configuration. To render a stored icon:

```tsx
import { IconRenderer } from '@/components/icon-picker';

// From database or config
const savedIcon = entity.icon; // "Activity"

<IconRenderer icon={savedIcon} className="size-5" fallback={<FallbackIcon />} />
```

<Aside type="caution" title="Rendering a saved name must not pull the whole barrel">
`IconRenderer` is the shared way to turn a single saved name into a component — use it for icons stored in **Structure** node types, admin JSON config, or anywhere a name arrives from the database or JSON. What such a renderer must **not** do is `import { icons } from 'lucide-react'` (the full barrel): that ships every icon into a hot, eagerly-loaded bundle — the exact cost the picker isolates in its lazy chunk. Lucide's documented pattern for icon "names stored in a database" is [`DynamicIcon`](https://lucide.dev/guide/packages/lucide-react#dynamic-imports) from `lucide-react/dynamic`, which code-splits one icon at a time:

```tsx
import { DynamicIcon } from 'lucide-react/dynamic';

// DynamicIcon takes kebab-case names, e.g. "alert-circle"
<DynamicIcon name="alert-circle" className="size-5" />
```

Mind the naming difference: this picker stores **PascalCase** names (`"AlertCircle"`), while `DynamicIcon` expects **kebab-case** (`"alert-circle"`) — so they are not drop-in interchangeable. Convert the name, or keep using a bundle-safe `IconRenderer` / a small curated `{ AlertCircle, … }` map when the set of possible icons is fixed.
</Aside>

<Aside type="note" title="Verify the renderer's internals against the code">
The export name (`IconRenderer`) and its `{ icon, fallback, strokeWidth }` contract are confirmed on this page, but **whether `IconRenderer` resolves names without importing the full barrel could not be verified from the docs repo**. Check the component source (`frontend/src/components/icon-picker/IconRenderer.tsx`): if it code-splits (via `lucide-react/dynamic` or a lazy lookup) it is safe to use anywhere; if it imports the full `icons` object, treat it as picker-chunk-only and use `DynamicIcon` / a curated map in hot render paths.
</Aside>

## Building a Custom Picker

For advanced use cases (category tabs, different layouts), compose from the headless primitives:

```tsx
import {
  useIconPicker,
  IconPickerGrid,
  CATEGORY_LABELS,
} from '@/components/icon-picker';
import type { Category } from '@/components/icon-picker';
import { Button } from '@/components/ui/button';
import { Input } from '@/components/ui/input';

function CustomIconPicker({ onSelect }: { onSelect: (icon: string | null) => void }) {
  const { search, setSearch, category, setCategory, categories, icons } = useIconPicker();

  return (
    <div>
      <Input
        value={search}
        onChange={(e) => setSearch(e.target.value)}
        placeholder="Search icons..."
      />

      {/* Category tabs */}
      <div className="flex flex-wrap gap-1 py-2">
        <Button
          variant={category === null ? 'default' : 'ghost'}
          onClick={() => setCategory(null)}
        >
          All
        </Button>
        {categories.map((cat) => (
          <Button
            key={cat}
            variant={category === cat ? 'default' : 'ghost'}
            onClick={() => setCategory(cat)}
          >
            {CATEGORY_LABELS[cat]}
          </Button>
        ))}
      </div>

      {/* Icon grid */}
      <IconPickerGrid
        icons={icons}
        search={search}
        onSelect={onSelect}
        className="grid grid-cols-[repeat(auto-fill,minmax(44px,1fr))] gap-2"
        renderIcon={(icon, onSelect) => (
          <Button
            key={icon.name}
            variant="secondary"
            onClick={onSelect}
            className="h-11 w-full px-0"
            title={icon.friendlyName}
          >
            <icon.Component className="size-6 shrink-0" />
          </Button>
        )}
        renderEmpty={() => (
          <p className="py-8 text-center text-muted-foreground">
            No icons found.
          </p>
        )}
      />
    </div>
  );
}
```
