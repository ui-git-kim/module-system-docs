This is the authoritative list of every component in the shadcn/ui registry, checked against the live site. It exists because **[Rule 8](/getting-started/llm-rules/#8-compose-the-shared-blocks-then-canonical-shadcn--never-hand-roll) says to compose canonical shadcn primitives** — and to do that, you (or an AI assistant) first have to know which primitives exist. The registry gains components regularly, so a list from memory is wrong within weeks.

<Aside type="danger" title="The live site is the only source of truth — cached lists lag">
When deciding whether a component exists, check **[ui.shadcn.com/docs/components](https://ui.shadcn.com/docs/components)** (or this page, which is verified against it). Do **not** trust:

- **The shadcn MCP `list_components` tool** — it returns a frozen **shadcn v4 snapshot of 46 components** and is missing `combobox`, `data-table`, `date-picker`, `calendar`, `popover`, `kbd`, `typography`, the whole AI/conversation set, and more. This is the usual reason an assistant insists "combobox doesn't exist" — **it does.**
- **shadcn's own `llms.txt`** — a curated subset that also lags (it omits `attachment`, `bubble`, `marker`, `message-scroller`, …).
- **Model training data** — the registry has changed shape (namespaced primitive variants) and grown well past what any snapshot captured.

Every link in the tables below was resolved live on **2026-07-19** — **64 components, all confirmed.**
</Aside>

## URL scheme — canonical page + three primitive namespaces

shadcn now ships each interactive component on top of a choice of primitive library. Every component has one **canonical page**, plus (for most) per-primitive variants:

| Path | What it is |
|------|-----------|
| `https://ui.shadcn.com/docs/components/<name>` | **Canonical page** — always use this to link/verify. Defaults to the Base UI variant and links out to the others. |
| `.../docs/components/base/<name>` | **Base UI** variant ([Base UI](https://base-ui.com/) primitives) |
| `.../docs/components/radix/<name>` | **Radix UI** variant (the classic primitives) |
| `.../docs/components/aria/<name>` | **React Aria** variant (Adobe's primitives) |

So `combobox` lives at [`/docs/components/combobox`](https://ui.shadcn.com/docs/components/combobox), and `/docs/components/radix/combobox` is the same component built on Radix. Pure-composition components (Card, Chart, Data Table, Typography, Input Group…) wrap no primitive and only have the canonical page.

<Aside type="note" title="Which variant does this template use?">
The starter's generated `components/ui/*` follow whatever the shadcn CLI writes for the app's `components.json` — you don't pick a namespace per component; you install with `npx shadcn@latest add <name>` and get the CLI's current default. The catalogue below always links the **canonical** page.
</Aside>

## What the template ships

`create-app` runs a single `npx shadcn@latest add …` with **33 primitives** (identical in `create-app.ps1` and `create-app.sh`). These arrive in `frontend/src/components/ui/` already themed:

```
sidebar breadcrumb separator dropdown-menu avatar collapsible badge card
dialog tooltip table checkbox sheet button input label radio-group switch
tabs alert alert-dialog empty spinner item native-select field command
input-group toggle toggle-group select skeleton textarea
```

They are marked <Badge text="in template" variant="success" size="small" /> in the catalogue below. Everything else is **add-on** — one command away.

A few add-ons already have their **npm dependency** pre-installed by `create-app`, so adding the component is friction-free: `sonner` (→ Sonner), `@tanstack/react-table` (→ Data Table), `date-fns` (→ Calendar / Date Picker).

<Aside type="caution" title="v1.62.0 baseline">
Starter **v1.62.0+** hosts ship these 33. A module can still land in an older host — so keep declaring every primitive your templates import in `shadcnComponents` (`src/module.ts`); there is no auto-detection. See [Rule 8](/getting-started/llm-rules/#8-compose-the-shared-blocks-then-canonical-shadcn--never-hand-roll).
</Aside>

## Adding a component

```bash
# In an app
cd frontend && npx shadcn@latest add combobox
```

```typescript
// In a module: declare it so components:sync installs it in the host
// src/module.ts
export const shadcnComponents = ['combobox', 'popover', 'calendar'];
```

After changing a module's `shadcnComponents` (or any `componentsJson`), run `npm run components:sync` in the host app so the union of module-declared primitives is (re)installed. See [Style System](/starter-template/style-system/#build-time-cold).

## Keeping this current

The registry moves faster than any static list. This page is refreshed by a **monthly scheduled agent** that re-verifies every component against the live site and opens a pull request when the set drifts (new components, renames, removals). The banner date at the top reflects the last verification.

If you need to verify something between refreshes, **fetch the component's page directly** — `https://ui.shadcn.com/docs/components/<name>` — rather than trusting a cached tool or memory.
