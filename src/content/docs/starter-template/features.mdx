## Quick Reference

### Frontend Imports

```typescript
// Authentication
import { useSession, getAuthToken, AuthProtectedRoute } from '@/features/auth';

// User Profile
import { useUserProfile, UserAvatar, UserProfileProvider } from '@/features/user';

// Admin Settings (global context)
import { useAppSettingsValue, AppSettingsProvider } from '@/features/admin';
```

### Backend Middleware

```typescript
import {
  requireAuth,    // Verify JWT token
  requireAdmin,   // Require admin role
  requireRole,    // Require specific role(s)
  attachRole,     // Attach role without requiring
} from '../middleware/auth.middleware.js';

// Protected route
router.use('/api', requireAuth, routes);

// Admin-only route
router.use('/admin', requireAuth, requireAdmin, adminRoutes);
```

### Backend Services

```typescript
// User service (via registry)
import { serviceRegistry } from '../../config/service.registry.js';
const userService = serviceRegistry.get('user');

await userService.getOrCreateProfile(userId, email);
await userService.updateProfile(userId, { displayName: 'Name' });
await userService.isAdmin(userId);

// Admin service
import { adminService } from './features/admin/admin.service.js';
await adminService.getAppSettings();
await adminService.updateAppSettings({ appName: 'My App' });
```

## How Features Connect

```
┌─────────────────────────────────────────────────────────────┐
│                        Dashboard                             │
│  ┌──────────────┐  ┌──────────────────────────────────────┐ │
│  │   Sidebar    │  │              Header                  │ │
│  │              │  │  ┌─────────────────────────────────┐ │ │
│  │  - Logo      │  │  │  HeaderToolbar (from registry)  │ │ │
│  │  - App Name  │  │  │  - Filtered by Admin overrides  │ │ │
│  │  - Nav Items │  │  └─────────────────────────────────┘ │ │
│  │    (filtered │  ├──────────────────────────────────────┤ │
│  │    by Admin) │  │              Content                 │ │
│  │              │  │  ┌─────────────────────────────────┐ │ │
│  │              │  │  │   User Profile / Settings       │ │ │
│  │              │  │  │   - Avatar, display name        │ │ │
│  │              │  │  │   - Preferences                 │ │ │
│  │              │  │  └─────────────────────────────────┘ │ │
│  └──────────────┘  └──────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    Auth (Neon Auth)                          │
│  - JWT tokens from JWKS endpoint                            │
│  - Email OTP sign-in                                        │
│  - Session management                                       │
└─────────────────────────────────────────────────────────────┘
```

**Data flow:**
1. **Auth** provides the JWT token for all API requests
2. **User** stores profile data (avatar, display name, preferences)
3. **Admin** stores app-wide settings (branding, overrides)
4. **Dashboard** renders the layout using settings from Admin and User

## Extending Features

All features use the registry pattern for extensibility:

```typescript
// Add navigation item
navigationRegistry.registerMain({
  id: 'my-feature',
  title: 'My Feature',
  url: '/my-feature',
  icon: MyIcon,
  order: 50,
});

// Add dashboard widget
dashboardRegistry.register({
  id: 'my-widget',
  title: 'My Widget',
  component: MyWidget,
  size: '2x1',
});

// Add settings page
settingsPagesRegistry.register({
  id: 'my-settings',
  title: 'My Settings',
  component: MySettingsContent,
  order: 60,
});

// Add header toolbar item
headerToolbarRegistry.register({
  id: 'my-action',
  type: 'action',
  position: 'right',
  icon: Bell,
  onClick: () => doSomething(),
});
```

See [Registry Reference](/starter-template/registries/) for the complete API.
