## Overview

Apps created with the setup script share git history with the template, enabling seamless updates via git merge.

---

## Git Remotes

| Remote | Purpose |
|--------|---------|
| `template-upstream` | Template repository for updates |
| `origin` | Your app's repository |

---

## Checking for Updates

```bash
npm run template:check

# Or manually
git fetch template-upstream
git log HEAD..template-upstream/main --oneline
```

---

## Applying Updates

<Steps>
1. **Ensure clean working directory**
   ```bash
   git status
   ```

2. **Fetch and merge**
   ```bash
   git fetch template-upstream
   git merge template-upstream/main
   ```

3. **Resolve conflicts** (if any)

4. **Test**
   ```bash
   npm run type-check
   npm run dev
   ```
</Steps>

---

## What Gets Updated vs Preserved

**Updated (accept changes):**
- `frontend/src/config/*.ts` - Registries
- `frontend/src/lib/*.ts` - Utilities
- `backend/src/config/*.ts` - Registries
- `backend/src/lib/*.ts` - Utilities
- `backend/src/middleware/*.ts` - Middleware

**Preserved (your code):**
- `frontend/src/features/*` - Your features
- `backend/src/features/*` - Your features
- `.env` files
- `prisma/schema.prisma`

---

## Handling Conflicts

```bash
# Keep your version
git checkout --ours path/to/file

# Take template version
git checkout --theirs path/to/file

# Abort if needed
git merge --abort
```
