The starter template is the base application that modules install into.

## What It Provides

- **App Shell** - Sidebar, header, layout components
- **Authentication** - Neon Auth integration with JWT
- **Registries** - Extension points for modules
- **Base Features** - User settings, preferences, dashboard
- **Role System** - User, admin, and managed roles

## Tech Stack

| Layer | Technology |
|-------|------------|
| Build | Vite + SWC |
| Frontend | React 19 + TypeScript |
| Styling | Tailwind CSS v4 |
| UI Components | shadcn/ui |
| Icons | Lucide (strokeWidth: 1) |
| Backend | Express.js + TypeScript |
| Database | Neon PostgreSQL + Prisma |
| Auth | Neon Auth |

## Base Features

### auth

Authentication pages and session management using Neon Auth. Provides:

- Sign in / sign out flows
- Email OTP verification
- JWT token management
- Session hooks (`useSession`)

### dashboard

App shell components that provide the application layout:

- Sidebar navigation
- Header with toolbar
- Content area with responsive padding
- Full-bleed layout option for builders/editors

### user

User management and personalisation:

- Profile settings (name, avatar, colour)
- Account preferences
- Language/locale selection
- Notification preferences

## Registries

The starter template provides registries for modules to extend:

| Registry | Purpose |
|----------|---------|
| navigation | Sidebar menu items |
| dashboard | Dashboard widgets |
| preferences | Settings sections |
| settings-pages | Settings page components |
| integrations | Third-party connections |
| header-toolbar | Header actions and toggles |

See [Registry Reference](/starter-template/registries/) for complete API documentation.
