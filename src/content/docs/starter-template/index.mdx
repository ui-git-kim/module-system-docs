## Philosophy

**Stop rebuilding the same foundation for every project.**

Every new application needs authentication, user profiles, settings pages, navigation, and a dashboard layout. Instead of recreating these from scratch each time, the starter template provides a solid foundation with well-defined extension points.

The idea is simple:
1. **Clone the starter** to get a working app shell with auth, users, and settings
2. **Install modules** that add the features you need (billing, teams, notifications, etc.)
3. **Build your custom features** using the same patterns and registries

Modules plug into the starter through **registries** - standardised extension points that ensure everything works together without conflicts. A billing module can add navigation items, settings pages, and API routes without modifying core files.

```
┌─────────────────────────────────────────────────────────┐
│                    Your Application                      │
├─────────────────────────────────────────────────────────┤
│   ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐   │
│   │ Billing │  │  Teams  │  │ Uploads │  │  Your   │   │
│   │ Module  │  │ Module  │  │ Module  │  │ Feature │   │
│   └────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘   │
│        │            │            │            │         │
│        ▼            ▼            ▼            ▼         │
│   ┌─────────────────────────────────────────────────┐   │
│   │              Registries (extension points)       │   │
│   │   navigation │ dashboard │ settings │ routes    │   │
│   └─────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────┤
│                    Starter Template                      │
│        Auth │ User │ Dashboard │ Admin │ Settings       │
└─────────────────────────────────────────────────────────┘
```

## What You Get

The starter template provides:

- **Authentication** - Neon Auth with JWT tokens, email OTP, and role-based access
- **User Management** - Profiles, avatars, preferences, and settings
- **App Shell** - Sidebar navigation, header toolbar, and responsive layouts
- **Admin Panel** - App branding, navigation overrides, and user management
- **Registry System** - Extension points for modules to plug into
- **Database** - Prisma 7 with Neon PostgreSQL (pooled connections)

Everything is TypeScript, fully typed, and follows consistent patterns.

## Registries

Modules extend the application through registries - each registry is an extension point for a specific part of the app:

| Registry | What It Extends |
|----------|-----------------|
| `navigationRegistry` | Sidebar menu items |
| `dashboardRegistry` | Dashboard widgets |
| `settingsPagesRegistry` | Settings page sections |
| `preferencesRegistry` | User preference options |
| `headerToolbarRegistry` | Header actions and toggles |
| `integrationsRegistry` | Third-party connections |
| `routeRegistry` | Backend API routes |
| `serviceRegistry` | Cross-feature communication |

Modules register their components at import time - no manual wiring required.

```typescript
// A module registers its navigation item
navigationRegistry.registerMain({
  id: 'billing',
  title: 'Billing',
  url: '/billing',
  icon: CreditCard,
  order: 40,
});

// It appears in the sidebar automatically
```

See [Registry Reference](/starter-template/registries/) for the complete API.

## Getting Started

```bash
# Clone the starter template
./create-app.sh my-app

# Install dependencies
cd my-app && npm install

# Set up your Neon database
# (see Neon Database Setup guide)

# Start development
npm run dev
```

Once running, you have a working app with:
- Sign-in page at `/sign-in`
- Dashboard at `/`
- User settings at `/settings`
- Admin panel at `/admin` (for admin users)

From here, install modules or build your own features.
