The create script (`create-app.ps1` on Windows, `create-app.sh` on macOS/Linux) sets up a new app interactively. This guide explains each prompt.

---

## Before You Start

Have these ready from your [Neon dashboard](https://console.neon.tech):

1. **Prisma snippet** - Go to Connection Details → Prisma tab → Copy snippet
2. **Auth URL** - From the Neon Auth section

:::tip[Credentials are optional]
You can skip the database and auth prompts during setup. The script creates placeholder comments in `.env` files that you can fill in later.
:::

## If the Folder Already Exists

If the target folder already exists, the script offers three choices before doing anything:

```
Directory 'C:\Users\you\ReactApps\my-saas-app' already exists.

  [O] Overwrite - Delete and recreate from scratch
  [U] Update    - Open the folder and use template:update instead
  [C] Cancel

Choose (O/U/C):
```

- **Overwrite** deletes the directory and recreates it. Before deleting, the script reads the previous `backend/.env` and offers to reuse the prior configuration (see below).
- **Update** prints the `npm run template:update` instructions for updating the existing app, then exits.
- **Cancel** exits without changes.

On **Overwrite**, the database, auth, and port prompts each show the previous value and offer to reuse it:

- `Reuse previous database config? (Y/n)` - reuses the prior `DATABASE_URL` / `DIRECT_DATABASE_URL`
- `Reuse previous auth URL? (Y/n)` - reuses the prior `NEON_AUTH_URL`
- The previous `PORT` becomes the default backend port

## Step 1: App Name

```
Enter your app name: my-saas-app
```

This becomes:
- Your app's display name
- Folder name (converted to kebab-case: `my-saas-app`)
- Package name in `package.json`

## Step 2: Output Location

```
Output Location
Where should the app be created?
Default: C:\Users\you\ReactApps
Output directory (press Enter for default):
```

The script creates the app in a sibling folder to the template by default. Press Enter to accept, or specify a different path.

## Step 3: Database URLs

The script asks you to paste the Prisma snippet from Neon:

```
Database Setup (Neon)
Get this from your Neon dashboard (https://console.neon.tech)
Go to: Connection Details -> Prisma tab -> Copy the snippet

Paste the ENTIRE snippet (both lines). It looks like:
DATABASE_URL="postgresql://...@ep-xxx-pooler.../neondb?..."
# DATABASE_URL_UNPOOLED="postgresql://...@ep-xxx.../neondb?..."

Or press Enter to skip (you'll need to configure .env manually later)

Paste your Neon snippet (press Enter when done):
```

### Getting the Neon Snippet

<Steps>
1. Go to [Neon Console](https://console.neon.tech)
2. Select your project
3. Click **Connection Details**
4. Select the **Prisma** tab
5. Make sure **Connection pooling** is enabled
6. Click **Copy snippet**
7. Paste the entire snippet into the terminal
8. Press Enter on a **blank line** to finish
</Steps>

:::note[Input is read line-by-line]
The script reads the snippet one line at a time until it sees an empty line. Pasting both lines and pressing Enter once leaves the prompt waiting for more input - press Enter again on a blank line to finish.
:::

The script automatically extracts both URLs:
- **Pooled URL** (with `-pooler`) - for app queries
- **Direct URL** (without `-pooler`) - for migrations

```
Parsed URLs:
  Pooled:  ...ep-xxx-pooler.ap-southeast-2...
  Direct:  ...ep-xxx.ap-southeast-2...
```

### Skipping Database Setup

Press Enter without pasting anything to skip. You'll need to:
1. Open `backend/.env` after setup
2. Uncomment and fill in `DATABASE_URL` and `DIRECT_DATABASE_URL`
3. Run `npx prisma migrate dev --name init`

## Step 4: Auth URL

```
Neon Auth Setup
Get this from Neon Console -> Auth section
Example: https://your-project-id.neonauth.ap-southeast-2.aws.neon.tech/neondb/auth

Or press Enter to skip (you'll need to configure .env manually later)
Enter your NEON_AUTH_URL:
```

Get this from your Neon project's Auth section. It's used for JWT token validation.

### Skipping Auth Setup

Press Enter to skip. You'll need to add `NEON_AUTH_URL` to both:
- `frontend/.env` as `VITE_NEON_AUTH_URL`
- `backend/.env` as `NEON_AUTH_URL`

---

## Step 5: Backend Port

```
Backend Port
Choose a unique port for this app's API server
Enter backend port (default: 3100):
```

Choose a unique port for this app's backend API. The default is 3100, but if you have multiple apps running, use different ports (3101, 3102, etc.).

The frontend port is handled automatically by Vite - it will use the next available port if 5173 is taken.

---

## Step 6: Prisma Schema Mode

```
Prisma Schema Mode
How should your database schema be organised?
  1. Single file (default) - All models in prisma/schema.prisma
  2. Multi-file - Separate .prisma files in prisma/schema/ directory
     (Recommended if you plan to install modules)
Enter choice (1/2, default: 1):
```

Choose how your database schema is organised. **Single file** (the default) keeps all models in `prisma/schema.prisma`. **Multi-file** splits models across separate `.prisma` files in a `prisma/schema/` directory - recommended if you plan to install modules, since each module can own its own schema file.

---

## Step 7: Vector Database (pgvector)

```
Vector Database (pgvector)
Enables AI embeddings and similarity search via pgvector
Adds the pgvector extension and a starter Embedding model
Enable vector database? (y/N):
```

Defaults to **No**. Answer `y` to enable AI embeddings and similarity search: the script adds the `pgvector` extension to your database and a starter `Embedding` model to the schema.

---

## Step 8: App Remote (Optional)

```
App Repository (Optional)
If you've created a GitHub repo for your app, enter the URL.
Or press Enter to skip (you can publish via VS Code later)
Example: https://github.com/yourusername/my-app
Enter your app's git remote URL:
```

If you've created a GitHub/GitLab repo for your app, enter the URL here. Otherwise, skip and set it up later via VS Code's "Publish Branch" feature.

Before doing any work, the script shows a summary:

```
===================================================
  Ready to create: my-saas-app
===================================================

  Location:     C:\Users\you\ReactApps\my-saas-app
  Backend port: 3100
  Schema:       Single file (prisma/schema.prisma)
  Vectors:      No
  Database:     Configured
  Auth:         Configured
  App remote:   None (local only)

Proceed? (Y/n):
```

Review the settings and press Enter (or `Y`) to proceed, or `n` to cancel.

The **Schema** line shows `Multi-file (prisma/schema/)` when you chose multi-file, and **Vectors** shows `Enabled (pgvector)` when enabled. If you skipped the database or auth prompts, their status reads `Manual setup required` instead of `Configured`.

## What Happens Next

The script runs in six labelled phases (`[1/6]` … `[6/6]`): clone, install dependencies, shadcn, env files, database, initial commit. Along the way it:

1. **Clones the template** with full git history (`[1/6]`)
2. **Sets up remotes** - `template-upstream` for updates
3. **Removes template-only files** (`create-app.ps1`, `create-app.sh`, `create-app.bat`)
4. **Removes the docs git submodule** - apps don't need the shared module-system docs
5. **Swaps in the app CLAUDE.md** - renames the bundled `CLAUDE.app.md` over the template's `CLAUDE.md`
6. **Updates package.json** with your app name
7. **Installs all dependencies** (frontend, backend) (`[2/6]`)
8. **Sets up shadcn/ui** and adds the template components (`[3/6]`)
9. **Creates .env files** with your credentials (or placeholders) (`[4/6]`)
10. **Runs Prisma generate** to create the database client (if configured) (`[5/6]`)
11. **Runs Prisma migrate** to set up database tables (if configured)
12. **Offers AWS KMS field-encryption setup** (optional) - prompts `Set up AWS KMS now? (y/N)`; if yes, runs `node scripts/setup-kms.mjs` (also available later as `npm run security:setup-kms`) to move field-encryption key custody to AWS KMS
13. **Creates an initial commit** (`[6/6]`)

## Environment Files Written

The `[4/6]` phase writes two `.env` files. Keys with a value are written live; the rest are written commented as placeholders for you to fill in later.

**`frontend/.env`**

| Key | Value |
|-----|-------|
| `VITE_NEON_AUTH_URL` | Your auth URL (commented if skipped) |
| `VITE_API_URL` | `http://localhost:<backend-port>` |
| `VITE_APP_NAME` | Your app name |
| `VITE_SECURITY_PROFILE` | `hardened` (match backend) |

**`backend/.env`**

| Key | Value |
|-----|-------|
| `DATABASE_URL` / `DIRECT_DATABASE_URL` | Pooled and direct URLs (commented if skipped) |
| `DATABASE_ADAPTER` | `neon` |
| `NEON_AUTH_URL` | Your auth URL (commented if skipped) |
| `PORT` / `NODE_ENV` / `FRONTEND_URL` | Server config |
| `SECURITY_PROFILE` | `hardened` |
| `SECURITY_EDGE` | `none` |

The backend file also ships **commented** placeholders for the optional security knobs: `SECURITY_ENCRYPTION_KEY`, `NEON_AUTH_PROJECT_ID`, `NEON_AUTH_SECRET_KEY`, `AWS_REGION`, `SECURITY_KMS_KEY_ID`, `SECURITY_KMS_WRAPPED_DEK`, `SECURITY_POSTURE_ENFORCE`, and `SECURITY_ADMIN_ALLOW_ENV_WRITE`. See [Configuration](/starter-template/configuration/) for what each does.

## Troubleshooting

### "Prisma migrate failed"

Check your `DATABASE_URL` and `DIRECT_DATABASE_URL` are correct. The pooled URL won't work for migrations - make sure you have the direct URL (without `-pooler`).

### "shadcn init failed"

You can run this manually later. The `init` step is interactive - **choose Radix when asked for the component library**:
```bash
cd frontend
npx shadcn@latest init
npx shadcn@latest add sidebar breadcrumb separator dropdown-menu avatar collapsible badge card dialog tooltip table checkbox sheet button input label radio-group switch tabs alert alert-dialog --yes --overwrite
```

### Script exits unexpectedly

The script keeps the window open on errors. Check the error message and verify your inputs.

### Can't parse Neon snippet

Make sure you're copying the complete snippet from the Prisma tab, including both lines (the pooled URL and the commented unpooled URL).

## Git Structure After Setup

Your new app has:

| Remote | Points To | Push? |
|--------|-----------|-------|
| `template-upstream` | Local template folder | No (disabled) |
| `origin` | Your app repo (if provided) | Yes |

To push to GitHub later:
```bash
git remote add origin https://github.com/you/your-app.git
git push -u origin main
```
