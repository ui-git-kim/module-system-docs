The create script (`create-module.ps1` on Windows, `create-module.sh` on macOS/Linux) sets up a new module project interactively from the module starter template.

---

## Before You Start

You need the following installed:

1. **Node.js** (v18+) and **npm**
2. **Git**
3. The **module starter template** cloned locally

---

## Running the Script

From the module starter template folder:

<Tabs>
  <TabItem label="Windows">
    ```powershell
    .\create-module.ps1
    ```
    Or double-click `create-module.bat`.
  </TabItem>
  <TabItem label="macOS/Linux">
    ```bash
    ./create-module.sh
    ```
  </TabItem>
</Tabs>

---

## Step 1: Module Name

<Tabs>
  <TabItem label="Windows">
    ```
    Module Name
    Examples: 'ai-chat', 'billing', 'My Feature', 'stripe_billing'
    (Will be converted to kebab-case automatically)

    Enter module name: My Cool Feature
    ```
  </TabItem>
  <TabItem label="macOS/Linux">
    ```
    Enter module name (e.g., 'ai-chat', 'billing', 'notifications'):
      - Use kebab-case (lowercase with hyphens)
      - This becomes the package name and CLI command

    Module name: my-cool-feature
    ```
  </TabItem>
</Tabs>

The name is used as the package name, CLI command, and folder name. On Windows, any input is automatically converted to kebab-case. On macOS/Linux, you must enter valid kebab-case directly.

| Input | Converted Name | Display Name |
|-------|---------------|--------------|
| `ai-chat` | `ai-chat` | Ai Chat |
| `My Feature` | `my-feature` | My Feature |
| `stripe_billing` | `stripe-billing` | Stripe Billing |

<Aside type="tip" title="Windows is more forgiving">
The PowerShell script accepts spaces, underscores, and mixed case — it converts everything to kebab-case. The bash script requires strict kebab-case input and rejects anything else.
</Aside>

---

## Step 2: Save Location

```
Save location (press Enter for default):
```

The default is the parent directory of the template folder. For example, if the template is at `~/ReactApps/unique-react-fullstack-module-starter`, the default save location is `~/ReactApps`.

The module folder is created inside the save location using the kebab-case name. If a folder with that name already exists, the script asks whether to overwrite it.

---

## What Happens Next

The script runs six automated steps:

1. **Clone template** — Git clones the template into the new folder, preserving full history. On macOS/Linux, falls back to `rsync` + `git init` if the clone fails.
2. **Configure git remotes** — Renames `origin` to `scaffold-upstream` and removes upstream tracking. This remote is used for pulling scaffold updates later.
3. **Remove scaffold-only files** — Deletes `create-module.ps1`, `create-module.sh`, `create-module.bat`, and `docs/ROADMAP.md`. Renames `CLAUDE.MODULE.md` to `CLAUDE.md`.
4. **Update package configuration** — Sets the package name, description, and `bin` entry in `package.json`. Updates `README.md` and `CLAUDE.md` with the module name.
5. **Update module.ts** — Replaces placeholder values (`my-feature`, `my_feature`, `My Feature`) with your module's kebab-case, snake_case, and display name variants.
6. **Install dependencies and commit** — Runs `npm install` and creates an initial git commit.

<Aside type="note" title="Windows-only verification">
The PowerShell script includes a verification step after the six main steps. It checks that key files (`package.json`, `src/module.ts`, `src/cli.ts`, template index files, and `update-scaffold.ps1`) exist in the new module.
</Aside>

## After Completion

The script displays a success message with next steps:

```
Success! Created ai-chat

Location: C:\Users\you\ReactApps\ai-chat

Next steps:

  1. Open the new module folder:
     cd "C:\Users\you\ReactApps\ai-chat"

  2. Build the module:
     npm run build

  3. Edit your module:
     - src/module.ts (configuration)
     - src/templates/ (file templates)

  4. Test in a starter project:
     npm link
     cd ../your-starter-project
     npx ai-chat init

  5. Update scaffold (when scaffold gets updates):
     .\update-scaffold.ps1
```

Both scripts offer to open the project in VS Code. The PowerShell script also offers to open the folder in Explorer.

## Git Structure After Setup

Your new module has a single remote:

| Remote | Points To | Purpose |
|--------|-----------|---------|
| `scaffold-upstream` | Module starter template folder | Pull scaffold updates |

There is no `origin` remote. Add one when you are ready to push:

```bash
git remote add origin https://github.com/you/your-module.git
git push -u origin main
```

The scaffold-upstream relationship:

```
Module Starter Template (scaffold-upstream)
  ↓ npm run scaffold:update (pulls changes)
Your Module (ai-chat, billing, etc.)
  ↓ npm link + npx your-module init
Target App (starter template clone)
```

## Troubleshooting

### Git not found

The script requires git to be installed and available in your PATH. Install git from [git-scm.com](https://git-scm.com/) and restart your terminal.

### Permission denied (macOS/Linux)

Make the script executable first:

```bash
chmod +x create-module.sh
```

### npm install fails

You can skip this and run it manually later:

```bash
cd your-module-name
npm install
```

Check that you have Node.js v18+ installed with `node --version`.

### Clone fails

On macOS/Linux, the script automatically falls back to `rsync` + `git init`. On Windows, it falls back to file copy. If both fail, check that the template directory is a valid git repository.

---

## Manual Clone Alternative

If you prefer not to use the interactive script, you can clone manually:

```bash
git clone https://github.com/ui-git-kim/unique-react-fullstack-module-starter my-feature-module
cd my-feature-module
git remote rename origin scaffold-upstream
```

Then update `package.json` (name, description, `bin` command) and `src/module.ts` (id, name, description) by hand. The `bin` key in `package.json` must match your module name:

```json
{
  "name": "my-feature-module",
  "bin": {
    "my-feature-module": "./dist/cli.js"
  }
}
```
