## What is a Module?

A **module** is a self-contained package that adds functionality to apps built with the starter template. Modules use the registry pattern to integrate without modifying core files.

## Module Architecture

```
module-name/
├── frontend/
│   └── src/features/[module]/
│       ├── index.ts
│       ├── navigation.registration.ts
│       ├── components/
│       └── pages/
├── backend/
│   └── src/features/[module]/
│       ├── index.ts
│       ├── route.registration.ts
│       ├── service.registration.ts
│       └── [module].service.ts
├── prisma/
│   └── schema.additions.prisma
└── docs/
    └── INSTALLATION.md
```

---

## Key Concepts

### Self-Registration

Modules register via side-effect imports:

```typescript
// App entry point
import '@module-name/frontend';  // Triggers registrations
```

### Non-Destructive

Modules never modify core files - only add via registries.

## Version Bumping

When modifying templates or scaffold infrastructure, bump the version:

```bash
npm run release:patch   # Bug fixes (most common)
npm run release:minor   # New features (add changelog entry)
npm run release:major   # Breaking changes (add changelog entry)
```

See [LLM Rules](/getting-started/llm-rules/#2-version-bumping) for which files require bumps.
