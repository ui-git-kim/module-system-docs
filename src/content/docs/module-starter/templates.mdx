These templates are ready to copy into your module's `src/templates/` directory. Remember to use placeholder patterns — `My Feature` for titles/labels, `my-feature` for kebab-case, `MyFeature` for PascalCase, `myFeature` for camelCase, `my_feature` for snake_case, and `my feature` for lowercase prose. All patterns get replaced with the actual module name during installation. See [Template Placeholders](/module-starter/building-modules/#template-placeholders) for the full reference.

<Aside type="tip">
All templates should include the standard file header with `FILE:` and `PURPOSE:` comments.
</Aside>

<Aside type="caution">
**Backtick escaping:** Templates are template literal strings. If your generated code uses backticks (`` ` ``) or template expressions (`${}`), you must escape them: `` \` `` and `\${}`. This is the most common cause of syntax errors in generated files. See [Escaping Rules](/module-starter/building-modules/#escaping-backticks-and-template-expressions) for examples.
</Aside>

### Detail Page (with Breadcrumbs)

Pages that display a specific entity (e.g., a project, invoice, or user) should set breadcrumbs so the site header shows the correct trail (e.g., "Projects > My Project").

```typescript
// src/templates/frontend/pages.ts
export const DETAIL_PAGE_TEMPLATE = `// ========================================================================
// FILE: frontend/src/features/my-feature/pages/MyFeatureDetailPage.tsx
// PURPOSE: Detail page for a single my-feature item
// ========================================================================

import { useEffect } from 'react';
import { useParams } from 'react-router-dom';
import { useBreadcrumbs } from '@/components/layout';
import { Card, CardContent, CardHeader, CardTitle } from '@/components/ui/card';

export function MyFeatureDetailPage() {
  const { id } = useParams<{ id: string }>();
  const { setBreadcrumbs } = useBreadcrumbs();

  // TODO: Replace with your data fetching hook
  const item = { name: 'Loading...' };

  // Set breadcrumbs when data loads
  useEffect(() => {
    if (item) {
      setBreadcrumbs([
        { label: 'My Feature', path: '/my-feature' },
        { label: item.name },  // No path = current page (not a link)
      ]);
    }
  }, [item, setBreadcrumbs]);

  return (
    <div>
      <Card>
        <CardHeader>
          <CardTitle>{item.name}</CardTitle>
        </CardHeader>
        <CardContent>
          <p className="text-muted-foreground">
            Detail content for item {id}.
          </p>
        </CardContent>
      </Card>
    </div>
  );
}
`;
```

<Aside type="tip">
Settings and admin pages set breadcrumbs automatically from the registry. You only need `useBreadcrumbs()` in your own custom page components — typically detail/edit pages that show a specific entity name in the header.
</Aside>

### Frontend Service

Every frontend service must use the shared `api` client — never `fetch()` or a custom Axios instance. The `api` client automatically injects auth tokens and points to the correct backend URL.

```typescript
// src/templates/frontend/services.ts
export const SERVICE_TEMPLATE = `// ========================================================================
// FILE: frontend/src/features/my-feature/services/my-feature.service.ts
// PURPOSE: API service for my-feature
// ========================================================================
// Every method accepts an optional AbortSignal so callers can cancel
// in-flight requests. The hooks in ../hooks/useMyFeature.ts thread
// TanStack Query's signal through automatically. Do NOT call this
// service directly from components — use the hooks.
// ========================================================================

import { api } from '@/lib/api';
import type {
  MyFeatureItem,
  MyFeatureListOptions,
  MyFeatureListResponse,
  CreateMyFeatureItemInput,
  UpdateMyFeatureItemInput,
} from '../types';

interface RequestOptions {
  signal?: AbortSignal;
}

export const myFeatureService = {
  // Cursor-paginated list — returns { items, nextCursor, hasMore }, not an array.
  async getAll(
    options: MyFeatureListOptions & RequestOptions = {}
  ): Promise<MyFeatureListResponse> {
    const { signal, limit, cursor } = options;
    const response = await api.get<MyFeatureListResponse>('/api/v1/my-feature', {
      params: { limit, cursor },
      signal,
    });
    return response.data;
  },

  async getById(id: string, options: RequestOptions = {}): Promise<MyFeatureItem> {
    const response = await api.get<MyFeatureItem>(\`/api/v1/my-feature/\${id}\`, {
      signal: options.signal,
    });
    return response.data;
  },

  async create(
    input: CreateMyFeatureItemInput,
    options: RequestOptions = {}
  ): Promise<MyFeatureItem> {
    const response = await api.post<MyFeatureItem>('/api/v1/my-feature', input, {
      signal: options.signal,
    });
    return response.data;
  },

  async update(
    id: string,
    input: UpdateMyFeatureItemInput,
    options: RequestOptions = {}
  ): Promise<MyFeatureItem> {
    const response = await api.patch<MyFeatureItem>(\`/api/v1/my-feature/\${id}\`, input, {
      signal: options.signal,
    });
    return response.data;
  },

  async delete(id: string, options: RequestOptions = {}): Promise<void> {
    await api.delete(\`/api/v1/my-feature/\${id}\`, { signal: options.signal });
  },
};
`;
```

<Aside type="caution">
**Never use `fetch()` or `new axios.create()` in frontend templates.** The shared `api` client from `@/lib/api` automatically injects the user's JWT token on every request. Without it, your API calls will fail with 401 Unauthorised. See [Auth Feature — API Client](/starter-template/auth-feature/#the-api-client) for details.
</Aside>

### Data Hooks (TanStack Query)

The `QUERY_HOOKS_TEMPLATE` generates `hooks/useMyFeature.ts` — a thin per-operation wrapper around the service using the starter template's shared TanStack Query client. Components should consume these hooks rather than calling the service directly.

Key conventions:
- **Key namespacing** — uses `createModuleQueryKeys('my-feature')` from `@/lib/query-keys`. Module keys **must** be namespaced with the module ID to prevent cache collisions between modules.
- **No custom hook factories** — write `useQuery`/`useMutation` directly at each call site. See [Starter Template — Data Fetching](/starter-template/data-fetching/) for the rationale.
- **Signal threading** — TanStack Query's `signal` is passed through to the service's `AbortController` support so unmount/re-fetch cancels in-flight requests automatically.

```typescript
// src/templates/frontend/hooks.ts
export const QUERY_HOOKS_TEMPLATE = `// ========================================================================
// FILE: frontend/src/features/my-feature/hooks/useMyFeature.ts
// PURPOSE: Data hooks for my-feature
// ========================================================================

import { useMutation, useQuery, useQueryClient } from '@tanstack/react-query';
import { createModuleQueryKeys } from '@/lib/query-keys';
import { myFeatureService } from '../services/my-feature.service';
import type {
  CreateMyFeatureItemInput,
  MyFeatureItem,
  MyFeatureListOptions,
  MyFeatureListResponse,
  UpdateMyFeatureItemInput,
} from '../types';

const myFeatureKeys = createModuleQueryKeys('my-feature');

// Queries
export function useMyFeatureList(options: MyFeatureListOptions = {}) {
  return useQuery({
    queryKey: myFeatureKeys.key('list', options),
    queryFn: async ({ signal }): Promise<MyFeatureListResponse> =>
      myFeatureService.getAll({ ...options, signal }),
  });
}

export function useMyFeatureItem(id: string | undefined) {
  return useQuery({
    queryKey: myFeatureKeys.key('item', id ?? ''),
    queryFn: async ({ signal }): Promise<MyFeatureItem> =>
      myFeatureService.getById(id!, { signal }),
    enabled: Boolean(id),
  });
}

// Mutations
export function useCreateMyFeatureItem() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: (input: CreateMyFeatureItemInput) => myFeatureService.create(input),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: myFeatureKeys.key('list') });
    },
  });
}

export function useUpdateMyFeatureItem() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: ({ id, input }: { id: string; input: UpdateMyFeatureItemInput }) =>
      myFeatureService.update(id, input),
    onSuccess: (updated, { id }) => {
      queryClient.setQueryData(myFeatureKeys.key('item', id), updated);
      queryClient.invalidateQueries({ queryKey: myFeatureKeys.key('list') });
    },
  });
}

export function useDeleteMyFeatureItem() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: (id: string) => myFeatureService.delete(id),
    onSuccess: (_, id) => {
      queryClient.removeQueries({ queryKey: myFeatureKeys.key('item', id) });
      queryClient.invalidateQueries({ queryKey: myFeatureKeys.key('list') });
    },
  });
}
`;
```

<Aside type="tip">
**For infinite-scroll lists**, swap `useQuery` for `useInfiniteQuery` — the service's cursor pagination (`nextCursor`) plugs straight into `getNextPageParam`. An example is included as a commented-out block at the bottom of the generated file.
</Aside>

### Style Registration

Contribute runtime CSS tokens and/or shadcn `components.json` values via the styleRegistry. Runtime fields (`tokens`, `css`) apply immediately; build-time fields (`componentsJson`) require `npm run components:sync` in the host app.

```typescript
export const STYLE_REGISTRATION = `// ========================================================================
// FILE: frontend/src/features/my-feature/registrations/style.registration.ts
// PURPOSE: Register runtime CSS tokens and/or shadcn components.json values
// ========================================================================
// Runtime (hot): tokens + css apply immediately
// Build-time (cold): componentsJson is merged by 'npm run components:sync'
//
// Exclusive fields (only ONE module per app): componentsJson.style,
// componentsJson.tailwind.baseColor, componentsJson.tailwind.cssVariables
// ========================================================================

import { styleRegistry } from '@/config/style.registry';

styleRegistry.register({
  id: 'my-feature.theme',
  order: 50,                   // 0-100 scale — later entries win on overlap

  // Runtime layer
  tokens: {
    // '--primary': 'oklch(0.7 0.15 240)',
  },
  // css: \\\`@layer components { .my-feature-card { ... } }\\\`,

  // Build-time layer (requires 'npm run components:sync')
  // componentsJson: {
  //   // Exclusive (one per app):
  //   // style: 'new-york',
  //   // tailwind: { baseColor: 'neutral', cssVariables: true },
  //   // Additive (many contributors):
  //   aliases: { 'my-feature/blocks': '@/blocks/my-feature' },
  //   registries: { '@my-feature': 'https://registry.myfeature.com/{name}.json' },
  // },
});
`;
```

<Aside type="caution">

</Aside>

### Service

```typescript
// src/templates/backend/services.ts
export const SERVICE_TEMPLATE = `// ========================================================================
// FILE: backend/src/features/my-feature/my-feature.service.ts
// PURPOSE: Business logic for my-feature
// ========================================================================

import { prisma } from '../../lib/prisma.js';
import { NotFoundError } from '../../lib/errors.js';
import { createModuleLogger } from '../../lib/logger.js';
import { actionRegistry } from '../../config/action.registry.js';
import type {
  CreateMyFeatureInput,
  UpdateMyFeatureInput,
  MyFeatureListOptions,
  MyFeatureListResponse,
} from './my-feature.types.js';

const log = createModuleLogger('my-feature');

const DEFAULT_PAGE_SIZE = 50;
const MAX_PAGE_SIZE = 200;

// List endpoints return a narrow projection — full objects only via getById.
const LIST_SELECT = {
  id: true, name: true, status: true, createdAt: true, updatedAt: true,
} as const;

export const myFeatureService = {
  // Cursor-paginated — stable under concurrent inserts; relies on the
  // [userId, createdAt] index. Returns { items, nextCursor, hasMore }.
  async getAll(
    userId: string,
    { limit = DEFAULT_PAGE_SIZE, cursor }: MyFeatureListOptions = {}
  ): Promise<MyFeatureListResponse> {
    const take = Math.min(Math.max(1, limit), MAX_PAGE_SIZE);
    const rows = await prisma.my_feature_item.findMany({
      where: { userId },
      orderBy: { createdAt: 'desc' },
      take: take + 1, // one extra to detect the next page
      ...(cursor && { cursor: { id: cursor }, skip: 1 }),
      select: LIST_SELECT,
    });
    const hasMore = rows.length > take;
    const items = hasMore ? rows.slice(0, take) : rows;
    return { items, nextCursor: hasMore ? items[items.length - 1].id : null, hasMore };
  },

  async getById(id: string, userId: string) {
    const item = await prisma.my_feature_item.findFirst({ where: { id, userId } });
    if (!item) throw new NotFoundError('Item not found');
    return item;
  },

  async create(userId: string, data: CreateMyFeatureInput) {
    // Field encryption (hardened+): seal columns declared in encryptedFields with
    // encryptField() from '../security/index.js' before storing. See security.registration.ts.
    const item = await prisma.my_feature_item.create({ data: { ...data, userId } });
    log.info({ userId, itemId: item.id }, 'Item created');
    actionRegistry.doAction('my-feature.item.created', { userId, item }); // fire-and-forget — not awaited
    return item;
  },

  // Atomic ownership check + update in a single transaction.
  async update(id: string, userId: string, data: UpdateMyFeatureInput) {
    const item = await prisma.$transaction(async (tx) => {
      const existing = await tx.my_feature_item.findFirst({ where: { id, userId }, select: { id: true } });
      if (!existing) throw new NotFoundError('Item not found');
      return tx.my_feature_item.update({ where: { id }, data });
    });
    log.info({ userId, itemId: id }, 'Item updated');
    actionRegistry.doAction('my-feature.item.updated', { userId, item, changes: data });
    return item;
  },

  // Single round trip — deleteMany with the compound filter is idempotent.
  async delete(id: string, userId: string) {
    const result = await prisma.my_feature_item.deleteMany({ where: { id, userId } });
    if (result.count === 0) throw new NotFoundError('Item not found');
    log.info({ userId, itemId: id }, 'Item deleted');
    actionRegistry.doAction('my-feature.item.deleted', { userId, itemId: id });
  },
};
`;
```

import { Router, Request, Response, NextFunction } from 'express';

const log = createModuleLogger('my-feature:routes');

router.get('/', async (req: Request, res: Response, next: NextFunction) => {
  try {
    const { id: userId } = req.user!;
    // Pagination: ?limit=50&cursor=<lastItemId> — returns { items, nextCursor, hasMore }.
    const limit = req.query.limit ? Number(req.query.limit) : undefined;
    const cursor = typeof req.query.cursor === 'string' ? req.query.cursor : undefined;
    const result = await myFeatureService.getAll(userId, { limit, cursor });
    res.json(result);
  } catch (error) {
    next(error);
  }
});

router.get('/:id', async (req: Request, res: Response, next: NextFunction) => {
  try {
    const { id: userId } = req.user!;
    const item = await myFeatureService.getById(req.params.id, userId);
    res.json(item);
  } catch (error) {
    next(error);
  }
});

router.post('/', async (req: Request, res: Response, next: NextFunction) => {
  try {
    const { id: userId } = req.user!;

    res.status(201).json(item);
  } catch (error) {
    next(error);
  }
});

router.patch('/:id', async (req: Request, res: Response, next: NextFunction) => {
  try {
    const { id: userId } = req.user!;
    const item = await myFeatureService.update(req.params.id, userId, req.body);
    res.json(item);
  } catch (error) {
    next(error);

router.delete('/:id', async (req: Request, res: Response, next: NextFunction) => {

## Prisma Schema

<Aside type="caution">
Module schemas must be **self-contained** with **no `@relation` directives**. Use plain foreign keys with indexes instead.
</Aside>

```typescript
// src/templates/schema.ts
export const SCHEMA = `
// ========================================================================
// My Feature Models
// ========================================================================
// Self-contained — NO @relation to the User model (avoids requiring changes
// to User). Use plain foreign keys + indexes. To add referential integrity,
// add the relation fields to your own User model.

model my_feature_item {
  id          String   @id @default(cuid())
  userId      String
  name        String
  description String?
  status      String   @default("active")
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  // Supports "items per user, newest first" without an in-memory sort.
  @@index([userId, createdAt])

  // Security integration (see security.registration.ts):
  // - Field encryption: keep a sensitive column as String and store the
  //   encryptField() envelope in it; declare it in encryptedFields.
  // - Neon RLS: add tenantId String (+ @@index([tenantId])) and list it in
  //   the manifest's ownedModels. tenantId == userId in solo mode.
}

// User settings — separate table (recommended over editing the User model).
model my_feature_settings {
  id        String   @id @default(cuid())
  userId    String   @unique          // @unique already creates the index
  enabled   Boolean  @default(true)
  config    Json     @default("{}")
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
`;
```

### Why No Relations?

Modules are installed independently and may be uninstalled. Using `@relation` would:
1. Require modifying the `User` model (which modules shouldn't do)
2. Create migration dependencies between modules
3. Cause issues when uninstalling modules

Instead, use plain foreign keys with indexes for performance.
