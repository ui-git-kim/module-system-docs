## What is this?

A registry-based architecture for building React applications. Instead of monolithic apps, you build with:

1. **Starter Template** - The app foundation with registries, authentication, and shell
2. **Module Starter** - Template for creating installable modules
3. **Modules** - Self-contained features that plug into any app built from the starter

## The Problem

| Issue | Traditional Approach |
|-------|---------------------|
| Features scattered across codebase | Hunt through multiple folders |
| Adding features requires modifying core code | Touch many files for one feature |
| Hard to reuse features across apps | Copy-paste and hope |
| Inconsistent patterns | Every feature structured differently |

## The Solution

| Issue | Module System Approach |
|-------|----------------------|
| Features scattered | Each module is self-contained |
| Modifying core code | Modules register themselves via registries |
| Hard to reuse | Modules install via CLI into any app |
| Inconsistent patterns | Standardised module structure |

## How It Works

### Registries

Registries are the glue. Instead of hardcoding navigation items, dashboard widgets, or routes, the app reads from registries. Modules register their contributions.

```typescript
// Module registers itself
navigationRegistry.register({
  id: 'billing',
  title: 'Billing',
  url: '/billing',
  icon: CreditCard,
});

// App reads from registry
const navItems = navigationRegistry.getAll();
```

### Side-Effect Imports

Modules register themselves when imported:

```typescript
// In app entry point
import '@/features/billing';  // Triggers all registrations
```

### Three Layers

```
┌─────────────────────────────────────────────┐
│                  Modules                     │
│  (billing, ai-chat, content, analytics...)   │
├─────────────────────────────────────────────┤
│              Module Starter                  │
│     (template for creating modules)          │
├─────────────────────────────────────────────┤
│             Starter Template                 │
│   (app shell, registries, auth, database)    │
└─────────────────────────────────────────────┘
```

## Quick Start

### 1. Clone the Template

**Using the create script (recommended):**

```bash
# Windows (PowerShell)
./create-app.ps1 my-app

# macOS/Linux
./create-app.sh my-app
```

**Or manually:**

```bash
git clone https://github.com/ui-git-kim/unique-react-fullstack-starter.git my-app
cd my-app
```

### 2. Install Dependencies

```bash
npm run install:all
```

### 3. Set Up Neon Database

See [Neon Database Setup](/getting-started/neon-setup/) for detailed instructions.

### 4. Configure Environment

```bash
# Copy example files
cp frontend/.env.example frontend/.env
cp backend/.env.example backend/.env

# Edit with your Neon credentials
```

### 5. Run Database Migrations

```bash
cd backend
npx prisma migrate dev
```

### 6. Start Development

```bash
npm run dev
```

This starts both frontend (http://localhost:5173) and backend (http://localhost:3100).
