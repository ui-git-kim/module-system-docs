The create script (`create-app.ps1` on Windows, `create-app.sh` on macOS/Linux) sets up a new app interactively. This guide explains each prompt.

## Before You Start

Have these ready from your [Neon dashboard](https://console.neon.tech):

1. **Pooled DATABASE_URL** - Has `-pooler` in the hostname
2. **Direct DATABASE_URL** - Same URL without `-pooler`
3. **NEON_AUTH_URL** - From the Auth section

## Running the Script

From the template folder:

```powershell
# Windows
.\create-app.ps1

# macOS/Linux
./create-app.sh
```

## Step 1: App Name

```
Enter your app name: my-saas-app
```

This becomes:
- Your app's display name
- Folder name (converted to kebab-case: `my-saas-app`)
- Package name in `package.json`

## Step 2: Output Location

```
Where should the app be created?
Default: C:\Users\you\ReactApps
Output directory (press Enter for default):
```

The script creates the app in a sibling folder to the template by default. Press Enter to accept, or specify a different path.

## Step 3: Template Source

```
Template Source
Where should the template be cloned from?

  1) Local - Clone from this local template (recommended for private repos)
  2) GitHub - Clone from a GitHub repository URL

Enter choice [1-2] (default: 1):
```

**Choose 1 (Local)** - This is currently the only supported option. The template is cloned from your local copy.

## Step 4: Database Type

```
Database Type
Select your database provider:
  1) Neon (recommended) - Uses WebSocket-based serverless adapter
  2) PostgreSQL (standard) - Uses TCP-based adapter (Supabase, Railway, local, etc.)

Enter choice [1-2] (default: 1):
```

**Neon** - Uses the `@prisma/adapter-neon` with WebSocket connections. Requires both pooled and direct URLs.

**PostgreSQL** - Uses the `@prisma/adapter-pg` with standard TCP. Works with Supabase, Railway, local PostgreSQL, etc.

## Step 5: Database URLs

### For Neon (Option 1)

```
DATABASE_URL (pooled): postgresql://user:pass@ep-xxx-pooler.region.aws.neon.tech/neondb
Enter your POOLED DATABASE_URL:
```

Enter the URL with `-pooler` in the hostname. This is used for app queries.

```
DIRECT_DATABASE_URL: postgresql://user:pass@ep-xxx.region.aws.neon.tech/neondb
(Same as above but WITHOUT '-pooler' in the hostname)
Enter your DIRECT DATABASE_URL:
```

Enter the same URL but remove `-pooler` from the hostname. This is required for Prisma migrations.

### For PostgreSQL (Option 2)

```
DATABASE_URL looks like: postgresql://user:pass@host/database?sslmode=require
Enter your DATABASE_URL:
```

Enter your single PostgreSQL connection string.

## Step 6: Auth URL

```
NEON_AUTH_URL looks like: https://your-project-id.auth.neon.tech
Enter your NEON_AUTH_URL:
```

Get this from your Neon project's Auth section. It's used for JWT token validation.

## Step 7: Backend Port

```
Enter backend port (default: 3100):
```

Choose a unique port for this app's backend API. The default is 3100, but if you have multiple apps running, use different ports (3101, 3102, etc.).

The frontend port is handled automatically by Vite - it will use the next available port if 5173 is taken.

## Step 8: App Remote (Optional)

```
App Remote Repository (Optional)
You can add a remote for your app's repository now, or skip and do it later.
Leave blank to skip.

Example: https://github.com/yourusername/my-app
Enter your app's git remote URL (or press Enter to skip):
```

If you've created a GitHub/GitLab repo for your app, enter the URL here. Otherwise, skip and set it up later via VS Code's "Publish Branch" feature.

## What Happens Next

The script automatically:

1. **Clones the template** with full git history
2. **Removes template-only files** (create-app scripts, etc.)
3. **Updates package.json** with your app name
4. **Installs all dependencies** (frontend, backend, shadcn)
5. **Creates .env files** with your credentials
6. **Runs Prisma generate** to create the database client
7. **Runs Prisma migrate** to set up database tables
8. **Creates an initial commit**

## After Completion

```
===================================================
  Success! Created my-saas-app
===================================================

Location: C:\Users\you\ReactApps\my-saas-app

Next steps:

  1. Open a new terminal
  2. cd "C:\Users\you\ReactApps\my-saas-app"
  3. npm run dev

Your app will be running at:
  Frontend: http://localhost:5173
  Backend:  http://localhost:3100

Open in VS Code? (Y/n):
```

## Troubleshooting

### "Prisma migrate failed"

Check your DATABASE_URL and DIRECT_DATABASE_URL are correct. The pooled URL won't work for migrations.

### "shadcn init failed"

You can run this manually later:
```bash
cd frontend
npx shadcn@latest init --yes --defaults
npx shadcn@latest add sidebar breadcrumb separator dropdown-menu avatar collapsible badge card dialog tooltip --yes
```

### Script exits unexpectedly

The PowerShell script keeps the window open on errors. Check the error message and verify your inputs.

## Git Structure After Setup

Your new app has:
- **No `origin` remote** by default (local only)
- **Full template history** preserved for updates
- **Initial commit** with your app name

To push to GitHub later:
```bash
git remote add origin https://github.com/you/your-app.git
git push -u origin main
```
