This page documents notable changes to the **Document Management** module. The module follows semantic versioning — **major.minor.patch**.

- **Major** (1.0.0 → 2.0.0): Breaking changes requiring migration
- **Minor** (1.0.0 → 1.1.0): New features, backwards compatible
- **Patch** (1.0.0 → 1.0.1): Bug fixes, documentation updates

Each entry groups changes under **Added**, **Changed**, and **Security** so you can scan for what affects your installation.

<Aside type="tip" title="Connector, not storage">
Document Management V1 is a **renderer and connector**. It connects to external storage, browses, previews, and downloads files, and keeps lightweight document references that link to Structure nodes. The database stores references, metadata, and your organisation (favourites, tags, node links) — never file content. In-app file creation (Plate), versioning, workflow states, export, and the companion desktop app are planned for V2.
</Aside>

## v1.2.0 <Badge text="Current" variant="success" /> <Badge text="17 June 2026" variant="note" />

Storage-provider management moves to **user settings**, and the admin page becomes an **engine policy**.

### Changed

- **Providers are now managed per user on a settings page.** The provider CRUD UI moved from the admin panel to a dedicated **Document Storage** settings page (`settingsPagesRegistry`, `/settings/document-storage`). Each user manages their own connections — the backend was already `userId`-scoped, so no data moves. `AdminStoragePage` became `DocumentStorageSettingsPage`.
- **The admin page (`/admin/documents`) is now an engine policy.** Admins toggle which adapter engines (S3/R2, WebDAV, Local) users may configure (`DocumentEnginesAdminPage`).

### Added

- **`doc_app_settings` table** — an app-wide singleton (`id="app"`, `allowedAdapters String[]`) holding the engine policy. An empty policy means "all installed engines"; the policy can only narrow the set installed at init, never widen it.
- **`GET /settings`** (any authenticated user; returns `{ allowedAdapters, installedAdapters, selectableTypes }`) and **`PUT /settings`** (admin-only via `requireAdmin`) to read and set the policy.
- `createProvider` now rejects engines the policy disallows (`403`), and the providers API's `selectableTypes` is derived from the policy.

<Aside type="caution" title="Updating from 1.1.x">
Run `npx document-management update` to pull the new pages and endpoints, then `prisma migrate dev` to create `doc_app_settings`. The user provider page is now at `/settings/document-storage`; `/admin/documents` now hosts the engine policy.
</Aside>

## v1.1.3 <Badge text="17 June 2026" variant="note" />

Patch release fixing two backend boot failures on apps running the latest starter template (Express 5 with ESM dependencies). Both bugs prevented the backend server from starting at all.

### Fixed

- **`archiver` v8 ESM compatibility.** `archiver` 8 is now an ESM-only package that exposes named exports (`ZipArchive`, `TarArchive`, `JsonArchive`) and no longer provides a default export or the `archiver('zip', …)` factory. The bulk-download service now imports `{ ZipArchive }` and constructs `new ZipArchive({ zlib: { level: 9 } })`. Previously the backend crashed on boot with `The requested module 'archiver' does not provide an export named 'default'`.
- **Express 5 wildcard routes.** Express 5 uses `path-to-regexp` v8, where bare `*` wildcards are invalid — they must be named. The browse/file/preview/download routes now use `/:providerId/*splat`, and the controller reads the matched path from the decoded `req.params.splat` segment array (`splat.join('/')`) instead of `req.params[0]`. Previously the backend crashed on boot with `Missing parameter name at index 21: /browse/:providerId/*`.

<Aside type="caution" title="Updating from 1.1.2">
If you installed 1.1.2, run `npx document-management update` to pull these fixes, then restart the backend.
</Aside>

Install it with:

### Added

#### Storage adapters

Three storage adapters implement a common `StorageAdapter` interface (`connect`, `disconnect`, `testConnection`, `listFolder`, `getFileInfo`, `exists`, `download`, `upload`, `delete`, `move`, `copy`, `createFolder`, `deleteFolder`, plus optional `getPreviewUrl`, `getThumbnailUrl`, and `search`).

| Adapter | Backing store | SDK | Notes |
|---------|---------------|-----|-------|
| `S3Adapter` | S3 / Cloudflare R2 | `@aws-sdk/client-s3`, `@aws-sdk/s3-request-presigner` | Presigned URLs for preview and thumbnails |
| `WebDAVAdapter` | ownCloud, Nextcloud, NAS | `webdav` | Basic auth or bearer token |
| `LocalAdapter` | Node `fs` (development) | — | Restricted to the base dir; rejects `../` and absolute paths |

The `createAdapter(type, config, { readOnly })` factory injects the `readOnly` flag so write operations throw `ReadOnlyError` at the adapter level. All adapter SDKs are always installed regardless of your selection, so the generated app always type-checks.

```typescript
import { createAdapter } from './adapters/index.js';

const adapter = createAdapter('s3', providerConfig, { readOnly: provider.isReadOnly });
await adapter.connect();
const entries = await adapter.listFolder('/contracts');
```

#### Provider management

Full CRUD for storage providers, with encrypted credential storage and live connection testing.

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/providers` | List the current user's providers |
| POST | `/providers` | Create a provider (credentials encrypted before the DB write) |
| GET | `/providers/:id` | Fetch a provider (config redacted) |
| PUT | `/providers/:id` | Update a provider |
| DELETE | `/providers/:id` | Remove a provider |
| POST | `/providers/:id/test` | Test the connection and refresh status/capabilities |

Provider status is one of `connected`, `error`, `pending`, or `unavailable`. The `toProviderResponse` serialiser omits `config`, so encrypted credentials are never returned to the client.

#### File browsing

List and grid views with breadcrumb navigation, backed by the `FileBrowser`, `FileGrid`, `FileList`, and `ProviderSidebar` components and the `useFolder` / `useFileInfo` hooks.

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/browse/:providerId` | List the root folder |
| GET | `/browse/:providerId/*` | List a subfolder by path |
| GET | `/file/:providerId/*` | File info for a path |

When a provider is unreachable, the `FileBrowser` shows an **unavailable** card with a **Retry** button (which calls `POST /providers/:id/test`), while cached `doc_document` references still appear in the documents list.

#### Preview by MIME type

The `FilePreview` component renders inline previews based on the file's MIME type:

| Category | Extensions | Strategy |
|----------|------------|----------|
| Images | jpg, png, gif, webp, svg | Inline `<img>` with zoom |
| PDF | pdf | `react-pdf` |
| Text / code | txt, md, csv, js, ts, py, json, … | Code block (bytes fetched via `/download`, read as text) |
| Video | mp4, webm | HTML5 `<video>` |
| Audio | mp3, wav, ogg | HTML5 `<audio>` |
| Office / archives / other | docx, xlsx, pptx, zip, tar, gz, … | Download prompt + file icon (no inline preview in V1) |

```typescript
// GET /preview/:providerId/* returns JSON, not a stream:
//   { kind: "url", url }   — presigned providers (S3)
//   { kind: "inline" }     — client blob-fetches via /download with auth preserved
```

#### Upload, download, and bulk download

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/download/:providerId/*` | Stream bytes with `Content-Disposition: attachment` |
| POST | `/upload/:providerId` | Raw request body is the file; `?path=` names the destination (including filename) |
| POST | `/bulk-download` | Body `{ providerId, paths[] }` returns a zip stream (`archiver`) |

Uploads use the raw request body rather than multipart, and upload size is enforced on the byte stream — a counting transform aborts at `DOC_MAX_UPLOAD_SIZE` (default 50 MB) rather than trusting the spoofable `Content-Length` header.

#### Document references

Lightweight references to remote files, with favourites, tags, notes, and SHA-256 content hashing for integrity verification. References are stored in `doc_document` and never contain file content.

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/documents` | List references (filters: `limit`, `cursor`, `providerId`, `status`, `isFavourite`, `tag`) |
| POST | `/documents` | Create a reference (`contentHash` computed server-side) |
| GET | `/documents/recent` | Recently accessed references |
| GET | `/documents/favourites` | Favourited references |
| GET | `/documents/:id` | Fetch one reference |
| PUT | `/documents/:id` | Update tags, notes, favourite, status |
| DELETE | `/documents/:id` | Delete a reference |

The hooks `useDocuments`, `useDocument`, `useRecentDocuments`, `useFavourites`, `useCreateDocument`, `useUpdateDocument`, and `useDeleteDocument` wrap these endpoints.

#### Folder bookmarks

Pin frequently used folders for one-click navigation. These three endpoints are **additional** to the design spec's 22 (they are not part of the original 5 endpoint groups).

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/bookmarks?providerId=` | List bookmarks (optionally per provider) |
| POST | `/bookmarks` | Create a bookmark |
| DELETE | `/bookmarks/:id` | Remove a bookmark |

Backed by `doc_folder_bookmark` and the `useBookmarks`, `useCreateBookmark`, and `useDeleteBookmark` hooks.

#### Recent-access audit trail

Every meaningful interaction is recorded in `doc_recent_access` with an `action` of `view`, `download`, `upload`, `link`, or `unlink`. This table powers the recent-documents views and doubles as a chain-of-custody audit trail.

#### Search

Cross-provider search exposed through `GET /search?q=&providerId=&limit=` and the `useSearch` hook, delegating to each adapter's optional `search` capability.

### Security

The release ships the full **provider-delegated** security model. A request flows through layered protection:

<Steps>

1. **Cloudflare Zero Trust Access** — authentication, MFA, device posture, and audit logs (free for up to 50 users).

2. **Cloudflare Tunnel** — encryption in transit with no port forwarding.

3. **Neon PostgreSQL** — encryption at rest and in transit.

4. **App-layer AES-256-GCM** — storage credentials are encrypted with `DOC_ENCRYPTION_KEY` before the DB write and redacted from API responses.

5. **Storage provider** — provider-managed file security.

</Steps>

- **Credential handling** — credentials are encrypted before being written, redacted in responses (`toProviderResponse` omits `config`), never logged, and the encryption key lives only in env. They are **not** environment variables — each provider's credentials live encrypted in `doc_storage_provider.config`.
- **Proxied downloads** — file downloads are proxied through the backend; the frontend never receives long-lived credentials. S3 may hand the client a short-lived presigned URL for **preview only**.
- **Local adapter hardening** — the local adapter is restricted to its base directory and rejects `../` and absolute paths.
- **Stream-enforced upload limits** — upload size is enforced on the byte stream (a counting transform aborts at `DOC_MAX_UPLOAD_SIZE`), not just the spoofable `Content-Length`.
- **Read-only providers** — read-only providers reject writes at the adapter level (`ReadOnlyError`).
- **Integrity & audit** — `contentHash` (SHA-256) is computed server-side at reference creation, and `doc_recent_access` doubles as a chain-of-custody audit trail.

<Aside type="caution" title="Set DOC_ENCRYPTION_KEY before going live">
`DOC_ENCRYPTION_KEY` is a **required secret** environment variable — a 32+ character key for AES-256-GCM credential encryption. It is collected interactively at `init` (the scaffold cannot generate values at prompt time). Generate one with `openssl rand -hex 32` and keep it stable: rotating or losing it makes existing encrypted provider credentials unreadable.
</Aside>

### Changed

These are deliberate deviations from the original design spec, documented here for transparency. They affect generated output and how you configure the module.

<Tabs>
  <TabItem label="Adapter selection">

Adapter selection is a single **select** (`s3` | `webdav` | `local` | `all`), not a true multi-select, because the bundled scaffold (v1.10.2) has no multi-select prompt type and `{{#if}}` supports only truthiness plus single-literal string equality (no array membership). The runtime constant `ENABLED_ADAPTERS` in `document-management.config.ts` derives the active set; `all` enables all three.

```typescript
// document-management.config.ts
const SELECTED = '{{storageAdapters|s3}}';

export const ENABLED_ADAPTERS: AdapterId[] =
  SELECTED === 'all' ? ['s3', 'webdav', 'local'] : [SELECTED as AdapterId];
```

  </TabItem>
  <TabItem label="Always-installed SDKs">

All adapter SDKs are installed unconditionally (not gated on the selection), so the generated target app always type-checks regardless of which adapters you enabled.

  </TabItem>
  <TabItem label="Encryption key">

The design-spec `encryptionKey` install prompt with auto-generate is **not** a prompt — the scaffold cannot execute code at prompt time, and env-var defaults are written literally (no `{{}}` substitution in env defaults). Instead `DOC_ENCRYPTION_KEY` is a required secret env var collected interactively at `init`; generate it with `openssl rand -hex 32`. Only `storageAdapters` and `defaultRegion` feed `{{...}}` template variables.

  </TabItem>
  <TabItem label="Raw-body uploads">

Uploads use the raw request body with `?path=` (not multipart) to avoid depending on a specific multipart middleware. Size is enforced on the stream.

  </TabItem>
  <TabItem label="JSON preview endpoint">

The preview endpoint returns JSON (`{ kind: "url" | "inline" }`) so the client blob-fetches via `/download` with authentication preserved, rather than streaming inline.

  </TabItem>
</Tabs>

## Roadmap

V2 builds on the V1 connector foundation:

- **In-app creation** — author documents directly in the app with [Plate](https://platejs.org/).
- **Versioning** — track document revisions over time.
- **Workflow states** — drive documents through review and approval states.
- **Export** — render and export documents to common formats.
- **Companion desktop app** — a native companion for local-first workflows.
