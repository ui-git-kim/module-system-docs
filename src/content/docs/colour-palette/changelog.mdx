## 2.16.0 ‚Äî 2026-06-20

### Palette builder ‚Äî roles, generators, View sheet, interactive Profile, favourites & history

The Palette Workspace becomes a real builder: colours flow into palettes by role, the Playground and Palette Profile both generate and edit in place, and colours can be favourited and revisited. Host-verified in a starter-template app.

- **Roles into palettes** ‚Äî the Colour Profile holder is now a list of role-tagged entries (hero / accent / functional) plus a shortlist. The palette sheet groups pickers by role and offers **Open in Playground**, **Create draft**, and **add to an existing palette**. The legacy single-colour brand-save is removed.
- **Generator toolbox** (`palette-generators.ts`) ‚Äî OKLCH generation primitives: golden-angle sequencing, distinct / best-candidate sampling, harmony inference + tone-line, `harmonySetColours` (analogous / complementary / triadic / tetradic / monochromatic), and a `generatePalette` orchestrator. Drives the Playground's spacebar regenerate.
- **Playground** ‚Äî seven generation modes (Random, Distinct, Analogous, Complementary, Triadic, Tetradic, Monochromatic); editable hex per slot; a permanent role icon + role dropdown; seeds from the working colour; a **Save dialog** (name + type: brand / ui / game / illustration / web / marketing / data-viz / exploration); and a **History** panel that restores past generations.
- **Colour View sheet** ‚Äî a wide right-hand sheet (Analytics / Harmonies / Scales) opened by the eye icon on any swatch; its "+" inserts the chosen harmony or scale colour next to the source.
- **Interactive Palette Profile** ‚Äî drafts load into the workspace store and edit in place: role dropdown, lock, weight slider, shortlist swap, and View per colour; spacebar regenerate with undo; publishing keeps the published version read-only.
- **Favourites (database-backed)** ‚Äî a new `colour_favourite` table with its own backend service and routes; a `useFavourites` hook; heart toggles on the Playground, Colour Profile, and View sheet; and a header button (via `headerToolbarRegistry`) scoped to the `/colour` and `/palette` routes.
- **Workspace store widened** ‚Äî functionals, sub-palettes, reference palettes, context templates, changelog and a browsable history, with checkpoint / restore-snapshot and undo across all of them; legacy fractional weights are normalised on load.

The alts and changelog **data contracts** are in place (`PaletteAlts.custom[]`, extended changelog kinds, checkpoint/restore) but their dedicated UI tabs are not yet wired ‚Äî see the [Roadmap](/colour-palette/roadmap/).

See [Palette](/colour-palette/palette/).

## 2.12.0 ‚Äî 2026-06-19

Iterates the v2.8.0 scaffold into usable pages and migrates the legacy data.

- **Playground** rebuilt as a full-bleed, Coolors-style column layout that auto-seeds from the Colour Profile collection (5 colours by default); spacebar generates, locks survive, "+" adds between.
- **Library** moved to `/palettes` with whole-card-clickable navigation to the Profile, plus a one-time migration of legacy Brand/UI saves into the new `palette` table.
- **Palette Profile** redesigned around a palette strip over tabs (Analytics, Alts & Comparison, Sub-palettes, Examples).
- **Legacy Brand/UI library page removed** ‚Äî the `colour_palette_saved` table is kept for back-compatibility.
- **Route-order fix** ‚Äî the palette and favourites routers now register **before** the legacy `/colour` router so their sub-paths resolve (the legacy greedy `GET /:id` would otherwise swallow them).

## 2.7.0 ‚Äî 2026-05-01

Current release.

### Mixing Playground (new)

- **New Mix tab structure** ‚Äî replaced the 7 mode tabs (Pair gradient, Weighted pile, etc.) with 4 goal-oriented tabs: 2 Colour Mix, Multi Colour Mix, Neutrals, Tonal Scales.
- **15 mixing algorithms** compared side by side with the same inputs: OKLCH shorter/longer hue, K-M spectral (3 variants), RYB cube interpolation (Gossett & Chen 2004), OKLab cartesian, OKLab + chroma boost, harmony-mapped gradient, pigmentMix (3 containment strategies), harmonizeMix, neutralFromPair, K-M/OKLCH piles, Gurney hull, K-M/OKLCH complementary neutrals, tinted black scale.
- **SVG visual diagrams** per algorithm: hue wheel arcs, OKLab a,b scatter, L-C gamut cross-sections with cusp/boundary/construction lines, brightness bars.
- **Inline parameters** per algorithm panel ‚Äî each algorithm has its own controls right next to its output.
- **Perceptual steps research section** ‚Äî gamut width chart, source/destination gamut triangles, even vs optimal gradient comparison.
- **Chain-walk harmony gradient** ‚Äî walks through every 0.5-degree intermediate gamut using cusp-line projection, accumulates delta-E, resamples at equal perceptual intervals.
- **Hue distribution bias slider** on the OKLab + chroma boost algorithm for researching gamut-aware step spacing.
- **New utility**: `colour-ryb.ts` ‚Äî RYB cube trilinear interpolation.
- **New component**: `MixLab.tsx` ‚Äî the full playground component.
- **New docs page**: Mixing Playground ‚Äî documents all algorithms, the open perceptual steps research problem, and key files.

## 1.48.95 ‚Äî 2026-04-29

### Refactor

- **Harmony pipeline rewritten as cusp-line projection.** The previous five-stage pipeline (H-K corrections, warm-lift, neon bell, multi-pass refinement) has been replaced by a single-pass geometric construction. `harmonyMap` decomposes the source colour as `(t, r)` inside a perceptual sub-triangle (vertices: `A` at the threshold-curve √ó gamut-boundary intersection, `B` at the constant-chroma threshold √ó gamut-boundary intersection, and `cusp` at the gamut peak). It then reconstructs the destination using the same `(t, r)` in the destination's sub-triangle. The mapping is mathematically self-inverse: `harmonyMap(harmonyMap(x, 180), 180) === x` in float OKLCH, with hex round-tripping limited only by 8-bit channel quantisation.
- **New helper `findThresholdCurveEdge(h, loL, hiL, rising)`.** Binary-searches for the L value where the gamut boundary crosses the chromatic-threshold *curve* (which itself varies with L). Used to place vertex A on the rising side where the gamut narrows fast.
- **Green/cyan neon suppression.** A Gaussian chroma cap centred at H=165¬∞ (œÉ=35¬∞, peak strength 30%) pulls chroma back from the gamut boundary in the green-cyan band only, where sRGB's unusually wide gamut produced overwhelming neon results. Other hues are untouched.
- **Removed:** `CUSP_L_CEILING`, `warmBase` / `warmLiftL` / `warmLiftC`, `neonBell`, `neonRisk`, `coolnessSharp`, `hkDeficit`, the asymmetric Pass 5 reduction, and the dual-cusp pattern. The single-pass cusp-line projection subsumes all of these geometrically.
- **`harmonyMap` no longer calls `gamutMap3DShaded`.** The destination sits inside its own sub-triangle by construction, so containment is guaranteed without a final mapping step.

### Research section redesign

- The Harmonies tab's interactive **Research** view is now a hue slider (0‚Äì360¬∞ destination hue) instead of a relative angle offset, with three side-by-side gamut cross-sections ‚Äî Source, Raw bary, and Harmony ‚Äî at full width. The Harmony triangle shows the three construction lines (cusp‚ÜíQ, B‚Üíedge-A‚Üícusp, A‚Üíedge-cusp‚ÜíB) converging at the destination point, and the values panel surfaces the three preserved ratios (`%cusp`, `%B`, `%A`).

## 1.3.34 ‚Äî 2026-04-08

### Features

- Import brand colours from Coolors URLs, Adobe Color URLs, hex input, and CSS variable blocks
- Auto-assign brand colours to semantic UI roles (action, structure, emphasis, highlight, background, foreground, neutral)
- Generate all 34 shadcn/ui CSS variables in OKLCH colour space for both light and dark modes
- Tinted neutral generation using brand hue for cohesive surfaces
- WCAG 2.1 AA contrast validation for every surface/foreground pair (with APCA support via `apca-w3`)
- Refinement sliders: neutral warmth, brand saturation, surface contrast, dark mode brightness
- Save, load, activate, and delete palettes per user
- Side-by-side light/dark mode comparison view
- Chart colour generation at 72¬∞ hue intervals
- Standalone **Colour Schemes** and **Colour Mix** showcase pages
- Embeds as a tab on Structure project nodes via `nodeTabRegistry`
- Self-contained Prisma schema (`colour_palette_saved`) with no cross-module relations
- Public API mounted at `/api/v1/colour-palette` with full CRUD plus `/active` and `/:id/activate`
- `PaletteOutput` exported as the canonical theme contract for consumer modules
