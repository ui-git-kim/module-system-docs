## 1.3.34 — 2026-04-08

Current release.

### Features

- Import brand colours from Coolors URLs, Adobe Color URLs, hex input, and CSS variable blocks
- Auto-assign brand colours to semantic UI roles (action, structure, emphasis, highlight, background, foreground, neutral)
- Generate all 34 shadcn/ui CSS variables in OKLCH colour space for both light and dark modes
- Tinted neutral generation using brand hue for cohesive surfaces
- WCAG 2.1 AA contrast validation for every surface/foreground pair (with APCA support via `apca-w3`)
- Refinement sliders: neutral warmth, brand saturation, surface contrast, dark mode brightness
- Save, load, activate, and delete palettes per user
- Side-by-side light/dark mode comparison view
- Chart colour generation at 72° hue intervals
- Standalone **Colour Schemes** and **Colour Mix** showcase pages
- Embeds as a tab on Structure project nodes via `nodeTabRegistry`
- Self-contained Prisma schema (`colour_palette_saved`) with no cross-module relations
- Public API mounted at `/api/v1/colour-palette` with full CRUD plus `/active` and `/:id/activate`
- `PaletteOutput` exported as the canonical theme contract for consumer modules
