**Colour semantics** is the cultural & symbolic layer: what specific colours *mean* in specific cultures, religions and contexts ‚Äî and, most usefully for a global tool, **where the meaning inverts** across audiences. It is reference data the [Method panel](/colour-palette/method-panel/) draws on, not a per-palette property.

<Aside type="caution" title="Conventions, confidence-tagged ‚Äî and not yet citation-backed">
These are conventions people hold, not effects a colour produces. Copy reads "commonly associated with‚Ä¶", never "means" as if universal. Every record carries a **confidence** (`high` / `medium` / `low`) so folkloric or contested claims can be down-weighted, and **`source` is intentionally left empty** in the seed ‚Äî real citations are attached later, and nothing here is fabricated. A **native-reviewer pass** on the granular regional claims is required before they are surfaced to users (see the [Roadmap](/colour-palette/roadmap/)).
</Aside>

## The record

Each row is a `ColourSemanticRecord`, keyed to one of the 14 anchors (the 11 Berlin‚ÄìKay basic-colour terms plus **gold**, **silver**, **turquoise**):

```ts
{
  colourAnchor: BasicColour;      // 'red' | 'blue' | ‚Ä¶ | 'gold' | 'turquoise'
  hueLo?, hueHi?: number;         // approx OKLCH hue band; null for L/C-defined anchors
  scope: 'cross-cultural' | 'regional' | 'religious' | 'domain';
  context: string;                // 'China' | 'Islamic world' | 'Western/Anglophone' ‚Ä¶
  association: string;
  valence: 'positive' | 'negative' | 'mixed' | 'neutral' | 'context-dependent';
  confidence: 'high' | 'medium' | 'low';
  culturalCaution?: boolean;      // appears in the localisation-landmine table
  note?: string;
  source?: string;                // EMPTY until a real citation is attached
}
```

`culturalCaution` is the highest-value field for a global product: it flags the cases where shipping a palette to the wrong audience genuinely backfires ‚Äî a "clean, celebratory" **white** that reads as funereal in East Asia, a flattering **green** that is an insult in Chinese-speaking markets, a **red** that swings from luck to alarm to mourning.

## Storage & API

The `colour_semantics` table is **global, admin-curated reference data** ‚Äî no `userId`, no relations, a sibling to the Thesaurus tables. It follows the established admin pattern: **reads open** to any authenticated user, **writes admin-gated**.

| Method | Path | Access |
|--------|------|--------|
| `GET` | `/api/v1/colour/semantics` (`?anchor=` `?scope=` `?caution=true`) | Authenticated |
| `GET` | `/api/v1/colour/semantics/anchor/:anchor` | Authenticated |
| `POST` | `/api/v1/colour/semantics` | Admin |
| `PATCH` | `/api/v1/colour/semantics/:id` | Admin |
| `DELETE` | `/api/v1/colour/semantics/:id` | Admin |
| `POST` | `/api/v1/colour/semantics/seed` | Admin |

The seed ships **62 built-in records** across all 14 anchors and upserts idempotently on `(colourAnchor, context, association)`, so re-seeding never clobbers admin additions or edits.

## The anchor classifier

The lookup resolves a generated colour to its anchor with the **Berlin‚ÄìKay classifier** (`palette-bk-category.ts`), which uses **lightness and chroma, not hue alone** ‚Äî so brown (a dark, desaturated warm) is told apart from orange, pink (a light, soft red) from red, and the achromatic terms (black / white / grey) from chromatic ones. Gold and silver are metallic reads that can't be detected from OKLCH, so they exist as anchors for their records but aren't emitted by the classifier.

```ts
nearestBasicColour(oklch) ‚Üí BasicColour   // 'red' | 'brown' | 'pink' | 'white' | ‚Ä¶
```

## Culture localisation & cautions

The [Method panel](/colour-palette/method-panel/) has a **Culture** selector ‚Äî a short curated list (China & East Asia, India & South Asia, Western, Islamic world, Japan, Ireland, Thailand, Latin America & Catholic Europe), each matched to the relevant `context` records by keyword so the dropdown stays friendly instead of listing every granular context. Choosing a culture does two things:

- **Localises generation (Area 4)** ‚Äî a *soft* hue lean toward that culture's positive, hue-banded associations is folded into the panel's hue-bias layer. It nudges, it doesn't dictate.
- **Surfaces cautions (Area 5)** ‚Äî for each of the palette's colours, the lookup resolves its anchor and shows any `culturalCaution` records for that culture as a quiet, dismissible note (e.g. *"White is commonly associated with mourning, death, funerals"*). Non-blocking ‚Äî information, not a gate. With no culture set, a one-line hint lists any charged anchors in the palette.

<Aside type="note" title="Curation lives in code for now">
The culture grouping is keyword-matched in the frontend against the free-text `context` field. A richer structured `culture`/`region` field ‚Äî imported with source links, the way the [Thesaurus](/colour-palette/thesaurus/) ingests its data ‚Äî is on the [Roadmap](/colour-palette/roadmap/).
</Aside>
