The Colour Palette module ships with sensible defaults and asks for no interactive configuration during install. Tuning happens through the in-app refinement sliders and the `palette-config.ts` constants file.

## Environment Variables

The module does not require any module-specific environment variables. It uses the standard starter-template database connection.

| Variable | Required | Purpose |
|----------|:--------:|---------|
| `DATABASE_URL` | yes | Pooled Prisma connection used by `colour_palette_saved` queries |
| `DIRECT_DATABASE_URL` | yes | Direct connection for `prisma migrate dev` |

<Aside type="caution">
Never use `prisma db push` for this module — always run `prisma migrate dev` so the migration history stays consistent with the rest of the app.
</Aside>

## Prompt Variables

The module exposes **no interactive prompts** during `npx colour-palette init`. Everything is generated from the same set of placeholders:

| Placeholder | Replaced With | Example |
|-------------|---------------|---------|
| `my-feature` | Module ID (kebab-case) | `colour-palette` |
| `myFeature` | camelCase | `colourPalette` |
| `MyFeature` | PascalCase | `ColourPalette` |
| `my_feature` | snake_case | `colour_palette` |

If you fork this module and want to ask the user a question during install (e.g. default neutral warmth), add an entry to `prompts: []` in `src/module.ts`. See the [Building Modules](/module-starter/building-modules/) docs for prompt syntax.

## Third-Party Services

None. The module runs entirely client-side for colour generation. Saved palettes are stored in your own Postgres database.

## Config Constants

The single source of truth for tuning is `frontend/src/features/colour-palette/config/palette-config.ts`. Edit it after install to change defaults without touching utility code.

| Constant Group | Purpose |
|----------------|---------|
| `LIGHTNESS_STOPS` | The 11 OKLCH lightness values for the tonal scale (50–950) |
| `NEUTRAL_TINT` | Brand-hue tint amount applied to generated neutrals |
| `CHART_HUE_STEP` | Hue rotation between chart colours (default 72°) |
| `CONTRAST_TARGETS` | Minimum WCAG ratios per surface/foreground pair |
| `DARK_MODE_OFFSETS` | Asymmetric lightness offsets for dark mode generation |

The `DEFAULT_REFINEMENT_SETTINGS` constant in `types/index.ts` sets the starting values for the in-app sliders:

```ts
export const DEFAULT_REFINEMENT_SETTINGS: RefinementSettings = {
  neutralWarmth: 0,
  brandSaturation: 1,
  surfaceContrast: 1,
  darkModeBrightness: 0.145,
};
```

## Database Schema

The module installs one self-contained table — no relations to the User model.

```prisma
model colour_palette_saved {
  id               String   @id @default(cuid())
  userId           String
  name             String
  brandColours     Json     @default("[]")
  generatedPalette Json     @default("{}")
  source           String   @default("manual")
  isActive         Boolean  @default(false)
  createdAt        DateTime @default(now())
  updatedAt        DateTime @updatedAt

  @@index([userId])
  @@index([userId, isActive])
}
```

| Column | Type | Notes |
|--------|------|-------|
| `userId` | String | Foreign key to `User.id` — enforced via index, not relation |
| `brandColours` | Json | Array of `BrandColour` objects (hex, oklch, weight, roles) |
| `generatedPalette` | Json | Snapshot of `PaletteOutput` (light + dark) |
| `source` | String | One of `coolors`, `adobe`, `manual`, `css` |
| `isActive` | Boolean | At most one active palette per user |
