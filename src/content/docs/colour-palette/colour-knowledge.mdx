**Colour knowledge** is the per-colour meaning layer: what a colour *evokes* (its psychology) and *who already uses it* (well-known brands). Together with the cultural [Colour Semantics](/colour-palette/colour-semantics/) layer it forms the **Colour Knowledge** area of the [Thesaurus](/colour-palette/thesaurus/) ‚Äî the colour-side reference data, surfaced for the working colour on the profile **Analyse** tab, not a per-palette property.

<Aside type="note" title="Why it lives with the Thesaurus">
Colour knowledge is colour-side reference data, a companion to the Thesaurus' names and design-system tokens ‚Äî so it lives under the Thesaurus (Colour Names / Design Systems / **Colour Knowledge**), and the palette Method *consumes* it rather than owning it. The same data drives the per-colour card and, in a later phase, biases generation: the Method's **Impact axis** values are exactly the psychology dimensions used here, so picking Impact = *Trustworthy* maps onto the colours whose psychology is trustworthy.
</Aside>

<Aside type="caution" title="Tendencies, confidence-tagged ‚Äî not citation-backed">
Psychology associations are conventions/tendencies, not effects a colour produces ‚Äî the copy mirrors the Impact axis ("tends to feel‚Ä¶"), never "makes users feel". Every row carries a **confidence** (`high` / `medium` / `low`) so weak rows can be down-weighted, and **`source` is intentionally left empty** in the seed ‚Äî never fabricated. Brand colours are factual references (a brand's known colour), **not endorsements**.
</Aside>

## The record

Each row is a `ColourKnowledgeRecord`, keyed to one of the 14 Berlin‚ÄìKay anchors and discriminated by `kind`:

```ts
{
  key: string;                    // deterministic identity, e.g. 'psychology:blue:trustworthy'
  colourAnchor: BasicColour;      // 'red' | 'blue' | ‚Ä¶ | 'gold' | 'turquoise'
  kind: 'psychology' | 'brand';
  // kind = 'psychology'
  dimension?: string;             // an Impact-axis dimension: calm | trustworthy | energetic |
                                  //   bold | joyful | serious | luxurious | friendly (extensible)
  strength?: 'high' | 'medium' | 'low';
  valence?: 'positive' | 'negative' | 'mixed' | 'neutral';
  // kind = 'brand'
  brandName?: string;             // 'Coca-Cola'
  brandHex?: string;              // the brand's reference colour
  industry?: string;              // 'Beverage'
  note?: string;
  confidence: 'high' | 'medium' | 'low';
  source?: string;                // EMPTY until a real citation is attached
}
```

Psychology dimensions deliberately reuse the [Method panel](/colour-palette/method-panel/)'s **Impact axis** vocabulary so the two stay in lockstep ‚Äî one source of truth for "what blue feels like".

## Storage & API

The `colour_knowledge` table is **global, admin-curated reference data** ‚Äî no `userId`, no relations, a sibling to `colour_semantics` and the Thesaurus tables. Reads are open; writes are admin-gated. Identity is a deterministic `key` slug (e.g. `psychology:blue:trustworthy`, `brand:red:coca-cola`) so the seed upserts idempotently across both kinds without nullable composite-unique pitfalls.

| Method | Path | Access |
|--------|------|--------|
| `GET` | `/api/v1/colour/knowledge` (`?anchor=` `?kind=`) | Authenticated |
| `GET` | `/api/v1/colour/knowledge/anchor/:anchor` | Authenticated |
| `POST` | `/api/v1/colour/knowledge` | Admin |
| `PATCH` | `/api/v1/colour/knowledge/:id` | Admin |
| `DELETE` | `/api/v1/colour/knowledge/:id` | Admin |
| `POST` | `/api/v1/colour/knowledge/seed` | Admin |

The seed ships per-anchor **psychology** rows (mapped to the eight Impact dimensions) and a curated set of **brands** per colour family, and upserts idempotently on `key` so re-seeding never clobbers admin additions.

## The per-colour card

On the colour profile **Analyse** tab (and, by reuse, the slide-in **Colour View** sheet) a **Colour Knowledge** card resolves the working colour to its nearest Berlin‚ÄìKay anchor (`nearestBasicColour`, the same classifier the semantics layer uses) and shows three columns for that anchor:

- **Psychology** ‚Äî the Impact-axis dimensions the colour evokes, with strength + confidence.
- **Cultural semantics** ‚Äî the [Colour Semantics](/colour-palette/colour-semantics/) records (context + association + valence, cautions flagged).
- **Brands** ‚Äî well-known brands that use the colour family, each with its reference swatch.

When nothing is recorded for an anchor the card points the admin to seed the data.

## Admin

The data is curated under **Admin ‚Üí Colour Thesaurus ‚Üí Knowledge**, a tab alongside the thesaurus' names and design-system sources. Three sub-tabs ‚Äî **Psychology**, **Cultural**, **Brands** ‚Äî each seed the built-in rows and browse them grouped by anchor. This **replaces the former standalone `/admin/colour-semantics` page**: cultural semantics now lives in this consolidated Colour Knowledge area.

<Aside type="note" title="Psychology moved out of the name layer">
Colour psychology used to live as free-text fields on the thesaurus **group** (`psychologySummary` / `psychologyPositive` / `psychologyNegative`, per named colour). Those fields have been **removed** from `colour_name_group`; psychology is now per-anchor structured data in `colour_knowledge`. The group keeps only name-scoped editorial metadata (`meaning` / `attributes` / `origin`).
</Aside>
