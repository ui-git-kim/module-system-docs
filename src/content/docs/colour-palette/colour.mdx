This is the **colour engine** — the foundation the rest of the module is built on. Everything here is about understanding and creating individual colours: the single-colour **Colour Profile**, the admin **Thesaurus** of named colours, and the maths that turns one colour into harmonies, scales, gamut-safe variants, and mixes.

The [Palette Workspace](/colour-palette/palette/) consumes this engine — every colour that lands in a palette comes from a Colour Profile tab or one of the algorithms below.

<Aside type="note">
The Colour Profile and the Thesaurus are **built and shipping**. The mixing and perceptual-step parts of the engine are **actively being worked on** — see the [Mixing Playground](/colour-palette/mixing-playground/) and the [Roadmap](/colour-palette/roadmap/).
</Aside>

## The Colour Profile

The Colour Profile is the single-colour deep-dive: pick any colour and inspect, transform, and derive from it across a set of tabs. Each tab is an **independent engine** — it does one job well and can be reused by higher levels.

| Tab | Status | Purpose | Deep reference |
|-----|--------|---------|----------------|
| **Analyse** | ✅ Built | Colour-space readouts (OKLCH / hex / HSL), Munsell approximation, accessibility and contrast info | — |
| **Gamut** | ✅ Built | sRGB gamut-triangle visualisation; gamut-safe mapping via the module's own 3D neighbourhood search | [Gamut Mapping](/colour-palette/gamut-mapping/) |
| **Harmonies** | ✅ Built | Complementary, triadic, tetradic, split-complementary and analogous harmonies via barycentric gamut-triangle matching | [Harmony Map](/colour-palette/harmony-map/) |
| **Data Vis** | ✅ Built | Distinguishable categorical colour sets for charts and diagrams | — |
| **Scales** | ✅ Built | Shade/tint ramps and paint-mix tonal scales from a single anchor | [Tonal Scale](/colour-palette/tonal-scale/) |
| **Adjust** | ✅ Built | Fine L/C/H adjustment tools; also opens as a dialog when tuning a specific palette colour | — |
| **Mix** | 🔧 In progress | K-M spectral mixing, OKLCH/OKLab interpolation, gradient step distribution | [Mixing Playground](/colour-palette/mixing-playground/) |

<Aside type="tip">
From the palette levels, clicking a colour opens its **Colour Profile in a drawer/panel** rather than navigating away — the palette context is never lost. The profile is a tool you reach into, not a destination.
</Aside>

## The Colour Engine

Underneath the tabs is a set of pure, OKLCH-native algorithms. They each have a dedicated derivation page documenting the maths, the design history, and the tuning constants.

<CardGrid>
  <Card title="Harmony Map" icon="random">
    `harmonyMap` rotates a colour by N degrees while preserving its perceptual position inside the gamut, using cusp-line projection inside a perceptual sub-triangle. Powers all harmony generators. — [Read](/colour-palette/harmony-map/)
  </Card>
  <Card title="Gamut Mapping" icon="seti:css">
    `gamutMap3D` finds the nearest *vivid* in-gamut point by searching L, H and C together — never a naive chroma clip. Used everywhere a colour must be made gamut-safe. — [Read](/colour-palette/gamut-mapping/)
  </Card>
  <Card title="Tonal Scale" icon="list-format">
    `generateTonalScalePaintMix` derives lighter tints and darker shades from one anchor using a painter's-pot metaphor — asymmetric, gamut-aware. — [Read](/colour-palette/tonal-scale/)
  </Card>
  <Card title="Mixing Playground" icon="puzzle">
    A research tool comparing 15+ mixing algorithms (K-M spectral, RYB cube, OKLab, pigment mix) side by side, plus the open perceptual-step-spacing problem. — [Read](/colour-palette/mixing-playground/)
  </Card>
</CardGrid>

Shared primitives used across these:

- **K-M spectral engine** — Kubelka-Munk reflectance mixing for paint-simulation blends and neutral generation.
- **Cusp utilities** — Ottosson-based cusp-finding (`findCuspForHue`, `maxChromaAtLH`) for gamut geometry. The module ships its own; it does not depend on `@texel/color`.
- **Chromatic threshold** — a CAM16-eccentricity-based threshold for the chroma below which a colour reads as achromatic, used by the harmony construction.

## The Thesaurus

The **Colour Thesaurus** is an admin-global reference of named colours — like a dictionary that maps working hexes to names, alt names, design-system scale memberships (Tailwind, Material, Open Color, CSS, BS381, XKCD…), and editorial metadata (meaning, psychology, origin).

It powers the name readouts on the Analyse tab and the toolbar autocomplete, and it is **read-only for users, admin-writable**. The full data model, ingest rules, and import format live on one page:

- **[Thesaurus](/colour-palette/thesaurus/)** — the four-table model, idempotent ingest rules, derived views, and the canonical JSON import format with worked examples.

## How the Engine Feeds Palettes

The engine is the supplier; the [Palette Workspace](/colour-palette/palette/) is the consumer. Two concrete links (Workspace features are **Planned**):

- **Add to considerations.** Any swatch in the Harmonies, Data Vis, or Scales tabs can be pushed onto a palette slot's consideration shelf with a "+" action.
- **Add between.** The Playground's "add between" inserts the best intermediate colour between two neighbours by asking the gradient **step-distribution** algorithm for a single step (N=1) — the same maths being developed in the [Mixing Playground](/colour-palette/mixing-playground/).

See the [Roadmap](/colour-palette/roadmap/) for the current state of the mixing and step-distribution work.
