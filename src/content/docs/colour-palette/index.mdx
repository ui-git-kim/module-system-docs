The **Colour Palette** module is a standalone colour pipeline. You feed it brand colours (from a Coolors URL, Adobe Color URL, hex list, or CSS variables), and it analyses them in OKLCH, auto-assigns semantic UI roles, and generates all 34 shadcn/ui CSS variables — with tinted neutrals, chart colours, and WCAG-validated contrast — for both light and dark modes.

The module is intentionally self-contained. It depends on nothing, registers no cross-module relations in the database, and exposes its `PaletteOutput` type as the canonical contract for any other module that wants to consume a generated theme.

<Aside type="note">
The page component does not register a sidebar entry by default — instead it registers as a tab on project nodes via the Structure module's `nodeTabRegistry`. The standalone **Colour Schemes** and **Colour Mix** showcase pages are registered as their own sidebar items.
</Aside>

## Quick Install

```bash
npx colour-palette init
```

The CLI creates the `colour_palette_saved` table, generates the frontend feature directory, installs the required shadcn components, and wires up the registrations.

### Full Feature List

- Import brand colours from Coolors URLs, Adobe Color URLs, raw hex input, or CSS variable blocks
- Auto-assign brand colours to semantic UI roles (primary, secondary, accent, destructive, etc.)
- Generate all 34 shadcn/ui CSS variables in OKLCH colour space
- Tinted neutral generation using brand hue for cohesive surfaces
- WCAG 2.1 AA contrast validation for every surface/foreground pair
- Light and dark mode output with asymmetric lightness inversion
- Refinement sliders: neutral warmth, brand saturation, surface contrast, dark mode brightness
- Save, load, and activate palettes with full history per user
- Side-by-side light/dark mode comparison view
- Chart colour generation at 72° hue intervals
- Per-colour functional role tagging with multi-role support
- Standalone Colour Schemes and Colour Mix showcase pages

## Architecture

The module is consumed two ways: as an embedded **tab** inside Structure project nodes, and as **standalone showcase pages** in the sidebar.

┌────────────────────────────────────────────────────────────┐
│  Starter Template App                                      │

│  ┌──────────────┐   PaletteOutput  ┌──────────────┐       │

│  │   Palette    │                  │  Modules     │       │
│  │   Module     │ ◄─── tab slot ── │  (Structure, │       │
│  └──────┬───────┘                  │   Builder)   │       │
│         │                          └──────────────┘       │

│  ┌──────────────┐   ┌──────────────┐   ┌──────────────┐  │
│  │   culori     │   │  apca-w3     │   │   zustand    │  │
│  │  (OKLCH)     │   │  (contrast)  │   │   (store)    │  │

### Pipeline Stages

| Stage | Module | Purpose |
|-------|--------|---------|
| Import | `palette-import.ts` | Parse Coolors / Adobe / hex / CSS into a list of brand colours |
| Convert | `colour-convert.ts` | OKLCH ⇄ hex ⇄ HSL conversions via `culori` |
| Analyse | `colour-analyse.ts` | Surface hue, chroma ceiling, harmony detection, perceptual ordering |
| Assess | `palette-assessment.ts` | Suggest functional roles for each input colour |
| Generate | `palette-generator.ts` | Build the full `InterfacePalette` for both light and dark modes |
| Validate | `contrast-validator.ts` | Check every surface/foreground pair against WCAG 2.1 AA |

## Prerequisites

- **Node.js 18+** and a starter-template-based app
- **Prisma + Neon (or PostgreSQL)** with `DATABASE_URL` configured
- **No module dependencies** — this module is standalone
- The **Structure** module is optional but recommended (the palette can be embedded as a project tab)
