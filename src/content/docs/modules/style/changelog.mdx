## v1.14.11 — July 2026

**Current release.**

- **Feature (base-class merge, Gap 1b):** the class designer now consumes the builder's per-project [base-class manifest](/modules/builder/integration/) (`GET /api/v1/builder/projects/:id/component-set` → `set.baseClasses`, keyed by `data-slot`; builder v1.18.0). `computeClassMergeInfo` resolves the node's active `cva` variant, `twMerge`s the component's base classes against the user's classes, and emits **variant-group mismatch warnings** the merge can't resolve — e.g. `max-w-2xl` won't override a shipped `sm:max-w-sm` — surfaced in the Classes field with a one-click "use `sm:max-w-2xl`" fix.
- **Feature:** new `useBaseClassManifest` hook and `getProjectComponentSet` service. Degrades to designer-only merge when no manifest or matching `data-slot` is present. Completes the Gap 1b follow-up to Gap 1a (v1.14.8).

## v1.14.10 — July 2026

- **Change (docs accuracy audit):** audited the module docs against the code and corrected the drift — the **index** now documents three Prisma models (was two), six registries including `structureRegistry` (was "five"), `/api/v1/style` plus the shadcn-config history routes and `security.registration` (was `/api/v1/my-feature`), and the `tailwind-merge` runtime dependency; the **design** page dropped the stale `typographyPreset` references, documented the baked-in sidebar-accent CSS path (`generateExtensionsCSS`), and corrected the default-styling injection-guard wording.
- **Dependency:** removed the dead `culori` dependency from `module.ts` — unused since the palette generator was extracted to the [colour module](/colour-palette/integration/).

## v1.14.9

- **Cleanup:** removed the dead HSL converters (`oklchToHsl` / `formatHsl` / `hslToOklch` / `hslCssToOklchCss`) and the `HslColour` interface from `oklch-utils.ts` (122 lines). HSL conversion now lives in the separate colour module.

## v1.14.8

- **Feature (Gap 3 — data-slot default styling):** new `StyleConfig.defaultStyling` (`{ slot, properties, scope? }[]`) restyles base components project-wide, class-free, emitted as `[data-slot="…"] { … }` rules inside `@layer components` by both CSS emitters (guarded like custom variables). Authoring UI to follow.
- **Change (Gap 1a — class merge):** `styleStateToClasses` now runs its output through `tailwind-merge` (extended for custom theme tokens) so the designer's own conflicting utilities resolve deterministically — the custom class, appended last, wins its group.
- **Cleanup:** removed the dead colour generators (`generateShadeScale`, `generateAccessiblePair`) and the dormant hardcoded `THEME_PRESETS` / `ThemePresetSelector`; palette generation belongs to the separate colour module. Counterpart derivation (`generateDark/LightCounterpart`) and `resolveTheme` retained.

## v1.14.7 — July 2026

- **Feature (two-layer theme config):** splits a project's `StyleConfig` into a **shadcn-config (preset) layer** and a **custom colour-token delta** (`delta = cssVars − resolveTheme(config)`, derived — no schema change for the delta). Layer helpers `computePaletteDelta` / `rebaseCssVars` / `reseedCssVars` / `affectsPalette` live in `utils/css-generator.ts`.
- **Feature (rebase — ask per change):** changing a palette-affecting preset field (`baseColor` / `colorPreset` / `chartColor` / `menuAccent`) now **prompts per change** — keep / rebase / discard — instead of silently wiping hand-tuned tokens. With no delta the base reseeds silently as before.
- **Schema:** new `style_shadcn_config_version` table and `/projects/:nodeId/shadcn-versions` routes give the Shadcn Config tab its **own** preset-layer history, distinct from the Theme tab's combined snapshots — deduped, so a token-only save adds no snapshot.
- **Change:** `DEFAULT_STYLE_CONFIG` charts reseeded from `resolveTheme('neutral')` so a fresh project starts with an empty delta.

## v1.14.6 — July 2026

- **Change:** extracted the hardcoded `nodeTypes: ['projects']` into a single `PROJECT_NODE_TYPES` constant (`structure.registration.ts`), seeded from a new full-mode-only `projectNodeType` install prompt (default `'projects'`) and editable post-install, so the structure tabs and the `theme_config` mirror field can target whatever node-type slug(s) the builder uses.

## v1.14.5 — July 2026

- **Fix (custom-theme foundations):** hand-edited colour tokens now survive a reload — the `resolveTheme` re-seed effect compares against the saved config's palette key instead of clobbering on load (fixed in all three editors). Custom CSS variables now reach the preview iframe, and custom-var names/values are validated and sanitised against stored CSS-injection (they inject app-wide and into other users' builder previews).
- **Fix (backend integrity):** `setActiveAppTheme` scoped to app overrides (with a clean 404); `isGlobal` / `isActive` removed from the PATCH schema (publish/activate only via admin routes); the version-number race retries on P2002 instead of 500'ing.
- **Change (coherence):** `useProjectTheme.save()` invalidates the cache and dispatches `style:theme-saved` so the builder preview refreshes.
- **Fix:** gradient drag debounced; theme-import and preset-paste now surface errors instead of silently no-op'ing.

## v1.14.4 — July 2026

- **Fix:** cleared ~143 latent `tsc` errors in the generated `features/style` code (the module's template-only type-check never saw them). Rewrote the className-designer section types to the canonical field model the parser and generator use, and fixed the runtime no-ops that mismatch exposed — `BorderSection` (width/color/style/radius/shadow/ring) and `BackgroundSection` (colour) controls wrote fields the generator never read; `ThemePresetSelector` read `.light` off a `StyleConfig` instead of `.cssVars.light`; `PresetCodeBar` encoded a raw `StyleConfig` instead of mapping through `styleConfigToPreset`.
- **Change:** documents the full `theme_config` [builder-integration contract](/modules/style/) on the module's index page.

## v1.14.3 — July 2026

- **Feature (shadcn create data sync):** new `npm run sync:shadcn` script fetches shadcn's canonical sources at a pinned tag (`shadcn@4.13.0`: `preset.ts`, `themes.ts`, `defaults.ts`, `font-definitions.ts`, `config.ts`) and regenerates one synced file, `utils/shadcn-data.ts` — the single source of truth for the preset codec arrays, base/accent OKLCH colour data, per-style defaults and font maps. Replaces three hand-copied, drifting snapshots.
- **Fix:** base swatches were near-identical greys → now distinct per-tone hues; base tones leaked into the Theme dropdown → the picker now shows the selected base tone plus accents (per shadcn's `getThemesForBaseColor`); adds the `rhea` style (with preview CSS); drops `gray`'s dead colour data (codec-only upstream); corrects stale accent OKLCH values. `theme-constants` folded into `shadcn-data`.

## v1.14.2 — July 2026

- **Schema:** added `@default(now())` to `style_theme_version.updatedAt` in `schema.ts` so the column introduced in v1.14.1 can migrate onto tables that already hold rows (Prisma can't add a bare required column to a populated table).

## v1.14.1 — July 2026

- **Feature (standards alignment + security):** scaffold → 1.15.0; the generated backend now ships `security.registration.ts` (manifest + idempotent deletion handler that fails loudly on tenant-offboard); ownership checks, transactional deletes and typed errors on the activate/restore/label routes; admin-only installs no longer emit builder/structure files (scaffold v1.14 `when:` gating).
- **Schema:** `schema.modelNames` completed; `style_theme_version.updatedAt` and a compound `@@index([userId, createdAt])` added.
- **Feature (builder contract):** the project theme now mirrors onto `node.data.theme_config` via a `structureRegistry` field plus a backend write; all seven shadcn styles present (adds `luma` / `sera`); `AbortSignal` threading; colour literals replaced with semantic status tokens; shared `theme-constants`; `info` token support; page-blocks adoption; module README.

## v1.14.0 — July 2026

- **Feature:** project CSS wired into the builder's iframe preview via a `builder.canvas.globalCss` [filter registration](/modules/builder/integration/) (`builder-canvas.registration.ts`), refreshed on the `style:theme-saved` event.

## v1.13.2

- **Change:** the `typographyPreset` field and the admin Typography section were pulled; typography will be reintroduced as a dedicated App Typography admin tab once shadcn's approach is researched.

## v1.13.1

- **Feature:** the sidebar-accent override is now emitted in the generated `global.css`; the preview iframe gains Dropdown Menu and Select cards tagged with `.cn-menu-target`.

## v1.13.0

- **Feature (shadcn preset alignment):** `resolveTheme` matches shadcn `buildRegistryTheme` byte-for-byte (`menuAccent: 'bold'` mutates `--accent`, not `--sidebar-accent`); the preset codec is a line-for-line port of `shadcn/preset/preset.ts` with V1 + V2 support; runtime menu-colour binding (via `.cn-menu-target` / `data-menu-translucent`) for dropdowns; the sidebar-accent extension shipped as a catalogue entry; radius `RADII` values match shadcn (0.45 / 0.625 / 0.875 rem).
- **Change (terminology clarity):** the admin settings panel now separates Menu (dropdowns) from Sidebar (our extension), and the design doc explains the distinction.
