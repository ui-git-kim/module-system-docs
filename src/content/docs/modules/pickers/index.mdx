The **Pickers** module provides 10 fully-featured, drop-in picker components for your application. Each picker is independently usable, and all pickers can optionally load configuration from **admin-managed profiles** stored in the database — giving administrators centralised control over picker behaviour without code changes.

The module generates both frontend components and a backend API for profile management, following the starter template's registry pattern for seamless integration.

---

## Key Features

- **ColorPicker** — Canvas-based HSL picker with solid, gradient, and image fill modes, alpha channel, eyedropper, and preset swatches
- **IconPicker** — Lucide icon browser with search, 42 categories, and tag-based filtering
- **DatePicker** — Calendar date selection with month/year navigation
- **DateRangePicker** — Dual-calendar range picker with configurable preset ranges
- **TimePicker** — Hour/minute/period selection supporting 12h and 24h formats
- **TimezonePicker** — Searchable timezone dropdown grouped by region
- **ImagePicker** — Drag-and-drop image upload with preview and file validation
- **EmojiPicker** — Categorised emoji grid with search and skin tone support
- **FontPicker** — Google Fonts browser with live preview
- **AvatarPicker** — Composed picker combining upload, icon, and initials modes
- **Admin Profiles** — Named picker configurations managed through an admin page
- **Showcase Page** — Interactive demo of all pickers, accessible at `/pickers`

## Architecture

```
┌─────────────────────────────────────────────────┐
│  Frontend (React)                               │
│                                                 │
│  ┌──────────────────┐  ┌─────────────────────┐  │
│  │  10 Picker        │  │  Admin Page          │  │
│  │  Components       │  │  (Profile Manager)   │  │
│  └────────┬─────────┘  └──────────┬──────────┘  │
│           │                       │              │
│  ┌────────▼───────────────────────▼──────────┐  │
│  │  usePickerProfile hook                     │  │
│  │  (loads + merges config from API)          │  │
│  └────────────────────┬──────────────────────┘  │
│                       │                          │
└───────────────────────┼──────────────────────────┘
                        │ /api/v1/pickers/profiles
┌───────────────────────┼──────────────────────────┐
│  Backend (Express)    │                          │
│                       ▼                          │
│  ┌──────────────────────────────────────────┐   │
│  │  pickersService (CRUD for profiles)       │   │
│  └────────────────────┬─────────────────────┘   │
│                       │                          │
│  ┌────────────────────▼─────────────────────┐   │
│  │  Prisma — pickers_profile table           │   │
│  └──────────────────────────────────────────┘   │
└──────────────────────────────────────────────────┘
```

Each picker component accepts an optional `profile` prop. When provided, the `usePickerProfile` hook fetches that profile's configuration from the backend and merges it with sensible defaults. Prop-level overrides always take highest precedence.

## Prerequisites

- **Starter template** — Built from `unique-react-fullstack-starter`
- **No additional modules required** — Pickers is fully standalone
- **No environment variables** — No third-party services or API keys needed

## Installation

<Steps>
1. **Install the module**
   ```bash
   npx pickers init
   ```
   The CLI generates all frontend components, backend routes, and the Prisma schema addition.

2. **Run the database migration**
   ```bash
   cd backend && npx prisma migrate dev --name add-pickers
   ```

3. **Import the module** (if not auto-imported)
   ```typescript
   // frontend/src/main.tsx
   import '@/features/pickers';

   // backend/src/app.ts
   import './features/pickers/index.js';
   ```

4. **Start the dev server**
   ```bash
   npm run dev
   ```
   Navigate to `/pickers` to see the showcase page, or visit the admin panel to create profiles.
</Steps>

<Aside type="tip">
  The module registers its own sidebar navigation and admin page automatically via the `navigationRegistry` and `adminPagesRegistry`. No manual route or menu configuration is needed.
</Aside>
