The Pickers module requires **no environment variables** and no third-party service configuration. All configuration happens through **picker profiles** — named JSON configurations stored in the database and managed via the admin page.

---

## Picker Profiles

A **picker profile** is a named configuration object that controls the behaviour of every picker type. Profiles are created and managed by administrators through the admin panel at **Admin > Pickers Profiles**.

Each profile contains a `config` JSON field with optional keys for each picker:

```typescript
interface PickerProfileConfig {
  colorPicker?: ColorPickerConfig;
  iconPicker?: IconPickerConfig;
  datePicker?: DatePickerConfig;
  dateRangePicker?: DateRangePickerConfig;
  timePicker?: TimePickerConfig;
  timezonePicker?: TimezonePickerConfig;
  imagePicker?: ImagePickerConfig;
  emojiPicker?: EmojiPickerConfig;
  fontPicker?: FontPickerConfig;
  avatarPicker?: AvatarPickerConfig;
}
```

One profile can be marked as the **default**. Setting a new profile as default automatically unsets the previous one.

## Config Merge Order

When a picker renders, configuration is resolved in this order (later overrides earlier):

1. **Built-in defaults** — Hardcoded in `usePickerProfile.ts`
2. **Profile config** — Loaded from the database via the `profile` prop
3. **Prop overrides** — Individual props passed directly to the component

```typescript
// Profile provides defaults, but this specific instance
// overrides enableAlpha and adds custom presets
<ColorPicker
  value={color}
  onChange={setColor}
  profile="marketing-brand"
  enableAlpha={false}
  presetColors={['#ff0000', '#00ff00', '#0000ff']}
/>
```

## Database Schema

The module creates a single table:

```prisma
model pickers_profile {
  id          String   @id @default(cuid())
  name        String   @unique
  description String?
  config      Json     @default("{}")
  isDefault   Boolean  @default(false)
  createdBy   String
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  @@index([name])
  @@index([isDefault])
  @@index([createdBy])
}
```

<Aside type="note">
  The schema is self-contained — it does not add relations to the `User` model or any other existing table. The `createdBy` field stores the user ID as a plain string with an index.
</Aside>

## Prompt Variables

The Pickers module does not define any interactive prompts during `npx pickers init`. Installation is fully automatic with no user input required.

## shadcn/ui Dependencies

The module uses the following shadcn/ui components (installed automatically during `init`):

| Component | Used By |
|-----------|---------|
| `button` | All pickers |
| `popover` | ColorPicker, IconPicker, DatePicker, DateRangePicker, EmojiPicker, FontPicker |
| `input` | ColorPicker, IconPicker, TimePicker, search fields |
| `scroll-area` | IconPicker, EmojiPicker, FontPicker, TimezonePicker |
| `dialog` | IconPicker (dialog mode), ImagePicker |
| `tabs` | ColorPicker, Showcase page |
| `badge` | Showcase page |
| `separator` | Various pickers |
| `label` | All pickers |
| `tooltip` | ColorPicker, IconPicker |
| `select` | TimezonePicker, TimePicker |
| `switch` | Admin page |
| `card` | Admin page, Showcase page |
| `slider` | ColorPicker (alpha, hue) |
