## Known Issues

Issues discovered but not yet resolved.

| Issue | Severity | Where | Workaround |
|-------|----------|-------|------------|
| Registry service component mappings are incomplete | Low | `registry.service.ts` ‚Äî `getImportForType()` only maps common shadcn components | Extensible but needs more component mappings for full code generation coverage |
| Tree panel is a placeholder | Low | `TreePanel.tsx` ‚Äî needs a full tree view with expand/collapse and drag reorder | Use the canvas panel for tree navigation |
| No undo/redo support | Medium | Builder canvas ‚Äî no history stack for block tree changes | Save frequently; use browser back if needed |
| Icon picker limited to Lucide | Low | `IconPicker.tsx` ‚Äî only supports Lucide icons in the picker UI, despite configurable icon library for code output | Use string input for non-Lucide icon names |
| Data hooks bypass TanStack Query | Medium | All `use*` data hooks (`useProjectBlocks`, `useBlocksLibrary`, `useNativeComponents`, `useNativeComponentsAdmin`) use imperative `useState`/`useEffect` instead of `useQuery`/`useMutation` ‚Äî no cache sharing, deduplication, or scoped invalidation | Works functionally but diverges from the starter template's [data-fetching conventions](/starter-template/data-fetching/) |
| No styleRegistry contribution | Low | Module does not register with `styleRegistry` despite controlling iframe preview CSS and pristine shadcn component selection | No runtime impact ‚Äî CSS is injected directly via iframe postMessage and scaffold shadcn sync |

### From the v1.12.1 health audit

A full standards/production/performance/security audit of the generated templates surfaced the items below. Security entries are framed as the hardening required rather than as reproduction steps.

| Issue | Severity | Where | Workaround |
|-------|----------|-------|------------|
| Registry and admin mutation routes need role-based authorisation | Critical | Backend routes ‚Äî global-state writes (component palette, config, shared fields, builder style, icon library) currently run for any authenticated user | Do not expose the builder API to untrusted tenants until route authorisation lands; restrict at the reverse proxy |
| Block-to-project assignment needs ownership checks | Critical | `builder.service.ts` ‚Äî `createBlock`/`updateBlock`/`duplicateBlock` accept an arbitrary `projectId`; the public registry serves by `projectId` + name | Single-tenant / trusted-user deployments only until ownership scoping is added |
| Generated code output needs escaping | Critical | Frontend tree-walker and `registry.service.ts` interpolate block-tree strings into JSX/registry output without escaping | Treat block authors as trusted until output escaping is added |
| Preview iframe needs origin + sandbox hardening | Critical | Canvas preview compiles and executes block-derived code; postMessage handlers do not verify origin and the iframe is unsandboxed | Serve a strict `frame-ancestors` CSP on the app; do not embed the builder cross-origin |
| Client-supplied `isSystem`/ownership flags need server authority | Critical | `builder.service.ts` ‚Äî `isSystem` from the request body is written directly; system blocks are visible to all users | ‚Äî |
| Registry handle needs server-side uniqueness/reservation | Critical | Handle uniqueness and format are only pre-checked client-side before being persisted via the generic preferences path | ‚Äî |
| PropertyPanel violates the Rules of Hooks | Critical | `PropertyPanel.tsx` ‚Äî `useRef`/`useCallback` run after conditional early returns, crashing the panel when selection changes | None ‚Äî hoist the hooks above the guards |
| Field template editor drops input focus every keystroke | Critical | `AdminComponentsPage` ‚Äî `FieldEditor` is declared inside the parent render and remounts on each change | None ‚Äî hoist `FieldEditor` to module scope |
| Generated code fails a strict TypeScript build | Critical | `DndProvider.tsx` (updater passed to a value-typed setter) and `ProjectBlocksWidget.tsx` (optional `node` dereferenced) | None ‚Äî fix the prop types / guards |
| Drag-drop into a node's own subtree deletes the node | Critical | `useBuilder.ts` ‚Äî `moveNodeIn` removes the subtree then fails to re-insert | Avoid dropping a node inside itself |
| Public registry code generator emits empty components | Critical | `registry.service.ts` reads `node.props`, but saved trees store props flat on the node ‚Äî served blocks generate blank output | ‚Äî |
| Empty component palette on a fresh install | Critical | No seed path for `native_component`; core feature is unusable until components are imported | Import components via `POST /components/import-json` after install |
| Undeclared `pickers` module dependency | Critical | Generated code imports `@/features/pickers`, but `pickers` is not in `requiredModules` | Install the pickers module manually before first build |
| Registry route registered with empty `versions: []` | High | `route.registration.ts` ‚Äî undocumented mounting behaviour for the public `/r` routes | Verify registry URLs resolve after install |
| Global-block registry URLs return 404 | High | Registry index advertises global blocks at `/r/{slug}/{name}.json`, but the detail route only queries project-scoped blocks | Reference global blocks by their project |
| No `securityRegistry` manifest or user-deletion cleanup | High | Module owns user-scoped data but registers no security manifest/deletion handler ‚Äî user deletion orphans blocks, categories, collections | ‚Äî |
| No backend input validation on mutating routes | High | Request bodies are cast straight to typed inputs across ~12 handlers | Validate upstream until schema validation is added |
| Forbidden Tailwind classes in generated feature code | Medium | ~127 occurrences of typography-scale, font-weight, colour-literal and `!important` classes outside `components/ui/*` | Cosmetic; violates the styling conventions |
| Registry proxy fetches lack timeouts; list endpoint over-fetches | Medium | `getShadcnIndex`/`getShadcnComponent` have no request timeout; `getBlocks` selects full `data` trees | ‚Äî |
| Schema uniqueness ineffective on nullable columns | Medium | Compound unique constraints include nullable columns (NULLs are distinct in Postgres); public registry can hit name collisions | ‚Äî |
| Private registry responses cached publicly | Medium | Bearer-protected registry responses set `Cache-Control: public` | Disable shared caching in front of the registry |

<Aside type="caution">
The full audit report (151 findings) is tracked privately. This roadmap records the remediation surface, not exploitation detail.
</Aside>

## Planned Improvements

### Short-term (Next Version)

**Migrate Data Hooks to TanStack Query**
- Replace imperative `useState`/`useEffect` fetch patterns with `useQuery`/`useMutation`
- Namespace all query keys with `['builder', ...]` prefix per starter convention
- Enable scoped invalidation (`queryClient.invalidateQueries({ queryKey: ['builder'] })`)
- Remove hand-rolled loading/error state where TanStack Query already provides it
- *Context:* Scaffold v1.9.0 ships a data-hooks template aligned with the starter's `QueryClient`. Apply the same pattern to `useProjectBlocks`, `useBlocksLibrary`, `useNativeComponents`, `useNativeComponentsAdmin`. Keep optimistic updates where UX demands them via `setQueryData`.

**Undo/Redo Stack**
- Add undo/redo support for block tree changes
- Keyboard shortcuts: Ctrl+Z / Ctrl+Shift+Z
- History panel showing recent operations
- *Context:* The `useBuilder` hook manages block state ‚Äî add a history array alongside `block` state. Each `setBlock` call pushes to history. Limit to ~50 entries.

**Tree Panel Implementation**
- Full hierarchical tree view with expand/collapse
- Drag-and-drop reorder within the tree
- Right-click context menu (delete, duplicate, wrap)
- Visual indicators for containers vs. elements
- *Context:* `TreePanel.tsx` exists as a placeholder. Use a recursive tree component with `@dnd-kit/sortable` for reorder.

**Broader Structure Hook Integration**
- Listen to `structure.node.updated` to handle project renames (update display references)
- Listen to `structure.node.archived` to cascade archive state to blocks
- *Context:* Currently only listens to `structure.node.deleted` for cleanup and `structure.node.response` for metadata enrichment. The structure module fires 16+ action hooks that the builder could react to.

### Medium-term (Upcoming)

**Component Import/Export**
- Export blocks as standalone shadcn registry items
- Import from external shadcn registries
- Version tracking for imported components
- Conflict resolution for name collisions
- *Context:* The registry JSON format is already defined in `registry.service.ts`. Needs a frontend UI for browsing external registries and an import resolution flow.

**Builder Logic Module**
- Separate module (`builder-logic`) for adding interactivity
- Event bindings (onClick, onChange, etc.)
- State management integration
- Data source connections
- Conditional rendering rules
- *Context:* This will be a new module that depends on `builder`. It should extend the property panel with a "Logic" tab (order 20) and the code generation registry with event handler imports and prop transforms. Store event bindings alongside block data.

**Enhanced Code Generation**
- Complete import mappings for all shadcn components
- Support for custom component libraries
- Code formatting with Prettier
- Multiple output formats (React, Vue in future)
- *Context:* Currently `registry.service.ts` has a limited `shadcnComponents` map. Expand it to cover all components in the `native_component` table dynamically.

**Multi-Library Icon Picker**
- Extend `IconPicker` to support all configured icon libraries (Tabler, Phosphor, Remix, Hugeicons)
- Load icon sets dynamically based on `shadcn:icon-library` config
- *Context:* Currently the picker only renders Lucide icons. The icon library config affects code output but not the picker UI.

### Long-term / Ideas

**Builder Style Module**
- Separate module (`builder-style`) for visual styling
- Tailwind class picker with live preview
- Theme generator for consistent design tokens
- Style presets and templates
- Responsive style overrides per breakpoint
- *Context:* Should register a "Style" tab (order 30) in the property panel tab registry and use `transformProps` in the code generation registry to apply className modifications.

**Multi-Block Canvas**
- Work with multiple blocks simultaneously on a split canvas
- Cross-block drag-and-drop
- Collection-level editing with entry point management
- Shared component references across blocks

**AI-Assisted Building**
- Natural language to component tree conversion
- Property suggestions based on context and common patterns
- Layout recommendations based on content type
- Accessibility improvement suggestions

**Collaborative Editing**
- Real-time multi-user editing with cursor presence
- Conflict resolution for simultaneous edits
- Change attribution and activity feed

**Concurrent radix + base shadcn styles**
- Support the radix and base style libraries at the same time rather than a single global `shadcn:builder-style`
- Style-scoped pristine folders and per-project (or per-block) style selection
- *Context:* the `shadcn:component:{style}:{name}` cache is already style-keyed; the single-style constraint lives in the sync target folder and the global style config.

### Code Health (from the v1.12.1 audit)

Lower-severity cleanups surfaced by the health audit, grouped for a maintenance pass:

- **Dead code:** remove unused template files (`frontend/types.ts`, `frontend/routes.ts`) and the unexported `ADMIN_PAGE_COMPONENT_TEMPLATE`; drop unused imports/vars and DEV `console.*` in the generated CanvasPage.
- **Conventions:** fix import ordering (external ‚Üí `@/` ‚Üí relative) across generated files; replace forbidden Tailwind classes with semantic tokens; refresh `package.json` description/keywords off the scaffold placeholder.
- **Config declarations:** declare `envVariables` in `module.ts` for `VITE_REGISTRY_DOMAIN`; export `CONFIG_KEYS`; remove dead `native_component` indexes; drop published sourcemaps from the npm package.
- **Performance passes:** `React.memo` on canvas recursion components, memoise context values, add `staleTime` to the near-static native-components query, lazy-load heavy registration imports, and avoid bundling the full lucide icon set.
- **Registry hardening (defence in depth):** reserved-handle denylist, timing-safe token comparison, enumeration-safe 404s, batched component-import upserts, and request timeouts on shadcn proxy fetches.
