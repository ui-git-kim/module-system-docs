## v1.17.1 тАФ July 2026

**Current release.**

- **Fix (preview theme):** the project's theme now reliably drives the canvas preview. The `/builder/canvas` iframe boots the whole app, so the host app's `global.css` `:root` also lives in it; the injected project theme is now re-appended as the **last** stylesheet in `<head>` so it wins the cascade (equal-specificity `:root` тЖТ source order decides) instead of the preview falling back to the app's default colours. `buildPreviewCss` also now normalises `theme_config.cssVars` тАФ accepting keys with or without the `--` prefix, and either the `{ light, dark }` shape or a flat token map тАФ so the project's variables are always emitted as valid custom properties.

## v1.17.0 тАФ July 2026

- **Snapshot runtime renderer:** the preview now renders against the project's pinned [component snapshot](/modules/builder/integration/#per-project-component-snapshots) тАФ completing the v1.16.0 feature. `IframeCanvas` forwards the snapshot to the iframe (`foundry:component-set`); `CanvasPage` compiles each source with Sucrase (`imports` transform + automatic JSX runtime) and executes it through a `require()` shim that resolves relative imports to other snapshot files (or a synthesised `./utils` `cn`) and non-relative imports from a curated `externalScope` (`radix-ui`, the `@base-ui/react/*` subpaths, `class-variance-authority`, `clsx`, `tailwind-merge`, `lucide-react`, `cmdk`, `vaul`, `react-day-picker`, `react-resizable-panels`, `input-otp`, тАж). Compilation is lazy, memoised and cycle-guarded, and a source that fails to compile is skipped in isolation. **The static pristine map remains the fallback when no snapshot is pinned**, so projects on radix and base render side-by-side against their real library + style.
- **Dependencies:** the frontend now declares `radix-ui`, `cmdk`, `vaul`, `react-day-picker` and `input-otp` (the packages synced components import) so the runtime `externalScope` resolves at build.
- **Fix:** icon-import injection no longer corrupts components whose last import is multi-line (e.g. `command`) тАФ the injected import is now inserted ahead of the first import instead of splitting the last one.

## v1.16.0 тАФ July 2026

- **Import overhaul:** the component import now reports **created vs updated** (it upserts by type тАФ existing components are updated, new ones added). The Import tab gained an **Export catalogue** button, and the redundant Export/Import buttons were removed from the Native Components table toolbar (both live on the Import tab now).
- **Per-project component snapshots:** new `builder_component_set` table + backend/service/routes and a `useProjectComponentSet` hook тАФ each project vendors its own timestamped snapshot of the shadcn component sources for its library + style. A **Component Sources** panel in the project's Blocks tab drives **Sync / Check for updates (diff) / Re-sync / Update files (promote)**, scoped by project ownership. *(Rendering the preview against the snapshot тАФ the runtime component compiler тАФ lands in a follow-up, once the style module writes the project's config field.)*
- **Project theme plumbing:** the builder reads the project's config (theme) from the structure node's registered config field (`node.data.theme_config`, with fallbacks), decoupling it from the style module's API.

## v1.15.0 тАФ July 2026

- **Change:** the icon picker now uses the starter template's built-in picker

- **Change:** the icon picker now uses the starter template's built-in picker (`@/components/icon-picker`) instead of the separate `pickers` module. The builder's `IconPicker` wrapper keeps the same `{ value, onChange }` contract; clearing is handled by the picker's "None" option.
- **Dependency:** `pickers` is **removed from `requiredModules`** тАФ the builder no longer needs it installed. (The icon picker remains Lucide-only, as before.)

## v1.14.0 тАФ July 2026

- **Feature:** dedicated **Import** tab in the component admin page, modelled on the colour module's import UX. Paste a JSON payload or upload a file; the builder export shape (`{ components, sharedFields }`) and the shadcn-components.json shape (`{ imports, components }`) are detected automatically, with inline parse errors, a format summary, and a post-import results report (imported counts + per-item errors).
- Import now runs server-side in bulk (`POST /components/import` and `/components/import-json`), so a single malformed row no longer aborts the whole batch. The Components table's **Import** button opens the new tab.

## v1.13.0 тАФ July 2026

Full standards / production / performance / security audit remediation. Scaffold infrastructure updated v1.9.1 тЖТ v1.15.0 (adds the `check:templates` escaping guard, security integration, and template perf fixes).

### Security

- **Authorisation:** every app-wide builder mutation (native component CRUD/import, config, shared fields, builder style, icon library, pristine sync) now requires the admin role тАФ previously any authenticated user could rewrite shared state.
- **Ownership:** block create/update/duplicate verify the target project belongs to the caller, and a client-supplied `isSystem` flag is stripped for non-admins тАФ closing cross-user registry injection and privilege escalation.
- **Generated output:** block-tree strings interpolated into generated JSX and registry output are now escaped (the readable `"..."` form is kept for safe values); dynamic tag and icon names are validated as identifiers.
- **Preview sandbox:** the canvas preview rejects `postMessage` from any origin/source other than its same-origin parent тАФ closing arbitrary-code execution via an external framer тАФ targets outbound messages at the exact origin, and adds a `sandbox` attribute to the preview iframe.
- **Public registry:** constant-time token comparison, enumeration-safe `404`s, `no-store` caching on private responses, and a path-traversal guard on pristine component names.
- **Account deletion:** a `securityRegistry` manifest and deletion handler remove a deleted user's blocks, categories and collections.

### Fixes

- Resolved the PropertyPanel Rules-of-Hooks crash, the `DndProvider` and `ProjectBlocksWidget` strict-build TypeScript errors, the field-template editor dropping input focus on every keystroke, and drag-and-drop into a node's own subtree silently deleting it.
- The public registry now serves each block's already-generated code (it previously emitted empty components) and derives dependencies from the component catalogue; advertised global-block URLs resolve instead of returning `404`.
- A fresh install seeds a starter component palette and declares the `pickers` module dependency and the `REGISTRY_DOMAIN` environment variable, so `npx builder init` produces working, compilable code.

### Performance & cleanup

- Memoised the native-components context value and added a `staleTime` to the near-static catalogue query; added timeouts to the backend shadcn proxy fetches and input validation to backend mutation routes; removed dead code and stray debug logs.
