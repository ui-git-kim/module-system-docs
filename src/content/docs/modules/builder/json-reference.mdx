The Block Builder's component palette is a database table (`native_component`) of component definitions. You can **bulk import** those definitions from JSON — to seed a project, share a palette between environments, or version-control your components.

Import happens on the **Admin → Block Builder → Import** tab (paste JSON or upload a file), or via the API. The Import tab **auto-detects** which of the two supported formats you pasted, shows a summary, and reports created/updated counts.

<Aside type="note">
All import endpoints are **admin-only** and mounted under `/api/v1/builder`. Import is an **upsert keyed by `type`** — a component whose `type` already exists is **updated**, a new `type` is **created**. The result is `{ imported, created, updated, errors }`, where `imported = created + updated` and `errors` holds a per-row message for any component that failed (the rest still import).
</Aside>

<Aside type="caution">
**Duplicate palette items?** Because import upserts by `type`, entries with the **same `type`** collapse into one. But two entries with **different `type`s and the same `label`** appear as **two** palette items — the usual cause of an apparently duplicated component (e.g. `alert-dialog` vs `alertdialog`, or a stray second entry with the same label). De-duplicate by both `type` and `label` before importing.
</Aside>

## Two supported formats

| Format | Top-level shape | `components` is | Endpoint |
|--------|-----------------|-----------------|----------|
| **Builder export** | `{ components: [...], sharedFields?: {...} }` (a bare array also works) | an **array** of full component objects | `POST /components/import` |
| **shadcn-components.json** | `{ imports: {...}, components: {...} }` | an **object keyed by `type`** | `POST /components/import-json` |

The **Builder export** format is the one the **Export** button produces (round-trips cleanly — server-managed `id`s are stripped on re-import), and is the recommended format for hand-authoring. The **shadcn-components.json** format is a convenience shape that shares one import path across many components via named `imports` groups.

## Builder export format

```json
{
  "components": [
    {
      "type": "button",
      "category": "element",
      "label": "Button",
      "icon": "MousePointerClick",
      "jsxTag": "Button",
      "importPath": "@/components/ui/button",
      "library": "shadcn",
      "sortOrder": 10,
      "properties": [
        { "key": "label", "label": "Label", "type": "string", "defaultValue": "Button", "codePlacement": "inner" },
        {
          "key": "variant",
          "label": "Variant",
          "type": "select",
          "defaultValue": "default",
          "options": [
            { "label": "Default", "value": "default" },
            { "label": "Secondary", "value": "secondary" },
            { "label": "Outline", "value": "outline" },
            { "label": "Destructive", "value": "destructive" }
          ]
        },
        { "key": "size", "label": "Size", "type": "select", "defaultValue": "default", "options": [
          { "label": "Default", "value": "default" },
          { "label": "Small", "value": "sm" },
          { "label": "Icon", "value": "icon" }
        ] },
        { "key": "className", "label": "Class Name", "type": "string" }
      ]
    },
    {
      "type": "input",
      "category": "element",
      "label": "Input",
      "icon": "TextCursorInput",
      "jsxTag": "Input",
      "importPath": "@/components/ui/input",
      "selfClosing": true,
      "sortOrder": 20,
      "properties": [
        { "key": "placeholder", "label": "Placeholder", "type": "string" },
        { "key": "type", "label": "Type", "type": "string", "defaultValue": "text" }
      ]
    },
    {
      "type": "card",
      "category": "container",
      "label": "Card",
      "icon": "SquareStack",
      "jsxTag": "Card",
      "importPath": "@/components/ui/card",
      "isContainer": true,
      "allowedChildren": ["card-header", "card-content", "card-footer"],
      "sortOrder": 60,
      "properties": [
        { "key": "className", "label": "Class Name", "type": "string" }
      ]
    }
  ],
  "sharedFields": {
    "className": { "key": "className", "label": "Class Name", "type": "string" }
  }
}
```

<Aside type="tip">
The outer object is optional — you can import a **bare array** of component objects (`[ { "type": "button", ... } ]`). Wrap it as `{ "components": [...], "sharedFields": {...} }` when you also want to import reusable field templates (`sharedFields`).
</Aside>

### Component fields

| Field | Type | Required | Default | Description |
|-------|------|:--------:|---------|-------------|
| `type` | string | ✅ | — | Unique component key (e.g. `button`). **Import upserts by this value.** |
| `category` | string | ✅ | — | `container` \| `element` \| `layout` \| `composite` \| `child` \| `primitive` (see [note below](#category-values)) |
| `label` | string | ✅ | — | Palette display name |
| `jsxTag` | string | ✅ | — | React component name or HTML tag emitted in generated code (e.g. `Button`, `div`) |
| `library` | string | | `shadcn` | `shadcn` \| `primitive` \| `plate` \| `custom` |
| `description` | string | | — | Optional palette description |
| `icon` | string | | — | Lucide icon name shown in the palette |
| `importPath` | string | | — | Import source for generated code (e.g. `@/components/ui/button`); omit for HTML primitives |
| `parentType` | string | | — | Parent component `type` — set on child components (e.g. `card-header` → `card`) |
| `selfClosing` | boolean | | `false` | Emit as `<Tag />` with no children |
| `isContainer` | boolean | | `false` | Can hold child nodes on the canvas |
| `acceptsAny` | boolean | | `false` | Accepts any child type (otherwise limited to `allowedChildren`) |
| `allowedChildren` | string[] | | `[]` | Whitelist of child component `type`s |
| `defaultChildren` | array | | — | Child nodes auto-inserted when the component is dropped on the canvas |
| `properties` | array | | `[]` | Editable property fields (see below) |
| `codeStructure` | object | | — | Code-generation hints (e.g. a conditional wrapper) |
| `sortOrder` | number | | array index | Palette ordering within its category |

### Property fields

Each entry in a component's `properties` array is an editable field shown in the property panel and (optionally) emitted into generated code.

| Field | Type | Required | Description |
|-------|------|:--------:|-------------|
| `key` | string | ✅ | Prop name — becomes a JSX attribute, inner content, or is hidden (see `codePlacement`) |
| `label` | string | ✅ | Field label in the property panel |
| `type` | string | ✅ | `string` \| `number` \| `boolean` \| `select` \| `icon` \| `textarea` |
| `defaultValue` | string | | Default value; also used as the emitted value when the user hasn't set one |
| `placeholder` | string | | Input placeholder |
| `description` | string | | Helper text under the field |
| `options` | `{ label, value }[]` | | Choices for a `select` field |
| `codePlacement` | string | | `attribute` (default — a JSX attribute) \| `inner` (content between the tags) \| `hidden` (not emitted) |
| `codeAlias` | string | | JSX attribute-name override (e.g. `htmlId` → `id`) |
| `codeFormat` | string | | `default` \| `array` (wraps the value as `{[value]}`) |
| `invertValue` | boolean | | Emit a boolean value inverted |
| `isRequired` / `isUnique` | boolean | | Validation hints |
| `richText` | boolean | | Enable rich-text editing for the field |
| `classNameKey` | string | | Companion className prop key (used by `icon` fields) |

<Aside type="caution">
Because code generation falls back to a property's `defaultValue`, a field with `codePlacement: "inner"` and a `defaultValue` (e.g. Button's `label` default `"Button"`) will emit that text as the element's children even before the user edits it — e.g. `<Button>Button</Button>`.
</Aside>

### sharedFields

`sharedFields` is an optional object map of reusable property templates (`{ [name]: propertyDefinition }`) using the same shape as a property field. They let you define a field once and attach it to many components from the admin UI. Only present in the **Builder export** format.

### Complete component (every field)

A single component with all fields populated, for reference:

```json
{
  "type": "table-cell",
  "library": "shadcn",
  "category": "child",
  "label": "Table Cell",
  "description": "A single cell in a table row",
  "icon": "Square",
  "jsxTag": "TableCell",
  "importPath": "@/components/ui/table",
  "parentType": "table-row",
  "selfClosing": false,
  "isContainer": true,
  "acceptsAny": true,
  "allowedChildren": [],
  "defaultChildren": [
    { "type": "label", "props": { "label": "Cell" } }
  ],
  "properties": [
    {
      "key": "colSpan",
      "label": "Column Span",
      "type": "number",
      "defaultValue": "1",
      "placeholder": "1",
      "description": "How many columns this cell spans",
      "codePlacement": "attribute",
      "codeFormat": "default",
      "isRequired": false
    },
    { "key": "className", "label": "Class Name", "type": "string" }
  ],
  "sortOrder": 70
}
```

`defaultChildren` entries are either a bare `type` string (shorthand) or an object `{ "type": string, "props"?: {...}, "children"?: [...] }` — so `["label"]` and `[{ "type": "label", "props": { "label": "Cell" } }]` are both valid, and children can nest. `codeStructure` is an advanced object of code-generation hints (e.g. a conditional wrapper) and can be omitted.

## shadcn-components.json format

This shape keys `components` by `type` (an object, not an array) and shares import paths through named `imports` groups. Each component references a group via `importGroup`, and uses `parent`/`children` instead of `parentType`/`allowedChildren`.

```json
{
  "imports": {
    "ui-card": { "path": "@/components/ui/card", "components": ["Card", "CardHeader", "CardContent"] },
    "ui-button": { "path": "@/components/ui/button", "components": ["Button"] }
  },
  "components": {
    "button": {
      "category": "element",
      "label": "Button",
      "icon": "MousePointerClick",
      "jsxTag": "Button",
      "importGroup": "ui-button",
      "properties": [
        { "key": "label", "label": "Label", "type": "string", "defaultValue": "Button", "codePlacement": "inner" }
      ]
    },
    "card": {
      "category": "container",
      "label": "Card",
      "icon": "SquareStack",
      "jsxTag": "Card",
      "importGroup": "ui-card",
      "isContainer": true,
      "children": ["card-header", "card-content"]
    },
    "card-header": {
      "category": "container",
      "label": "Card Header",
      "jsxTag": "CardHeader",
      "importGroup": "ui-card",
      "parent": "card",
      "isContainer": true
    }
  }
}
```

### Field mapping

The `import-json` endpoint transforms this shape into the builder format before upserting, so the field semantics match the [component fields](#component-fields) above with these differences:

| shadcn-components.json | Maps to | Notes |
|------------------------|---------|-------|
| `imports[group]` → `{ path, components }` | — | Named import groups referenced by `importGroup` |
| `importGroup` | `importPath` | Resolved to `imports[importGroup].path` |
| `parent` | `parentType` | Parent component `type` |
| `children` | `allowedChildren` | Whitelisted child `type`s |
| `jsxTag` (optional) | `jsxTag` | Defaults to the component's `type` when omitted |
| _(all components)_ | `library: "shadcn"` | Forced to `shadcn` |
| _(object key order)_ | `sortOrder` | Assigned sequentially from key order |

The per-component fields `category`, `label`, `description`, `icon`, `isContainer`, `acceptsAny`, `selfClosing`, `properties`, `defaultChildren`, and `codeStructure` carry over unchanged.

<a id="category-values"></a>
<Aside type="note">
Valid `category` values are `container`, `element`, `layout`, `composite`, `child`, and `primitive`. The palette currently surfaces **Containers, Elements, Layouts, and Primitives** as groups (child components appear nested under their parent). `layout` and `composite` are still being fleshed out — see the [roadmap](/modules/builder/roadmap/).
</Aside>

## Preview rendering

Importing a component **definition** (its palette entry) does not, by itself, ship the component's **source code**. But as of v1.19.0 the **import automatically syncs the sources** too, so imported shadcn components render in the preview straight away. The live preview compiles generated JSX and resolves each component's `jsxTag` to a real implementation from, in order:

1. the selected project's **synced component snapshot** (the project's library + style); else
2. the app-wide **global component set** — populated by import; used for **global and system blocks and whenever no project is selected**; else
3. the built-in **pristine shadcn set** — the ~40 default components (accordion, alert, badge, button, card, table, …).

So a component like `attachment` — a real shadcn component — renders everywhere after import, including for global/system/no-project blocks, without pinning a project. A component whose `importPath` isn't a fetchable shadcn registry entry (a genuinely custom/app-local component) still shows as a dashed **"(preview unavailable)"** placeholder: the sandbox has no source to execute, though the **generated code is unaffected** — it still emits `<Attachment>` with your `importPath`, so the exported component works in a real app that ships that import.

<Aside type="note">
**How the sync happens.** On import, the builder derives the shadcn slugs from every component's `importPath` (`…/ui/<slug>`), fetches + rewrites their sources, and stores them in the global set (admin-only, `POST /components/global-set/sync`). Fetches are cached, and slugs that aren't fetchable are reported in the import result and skipped. Selecting a project and running **Component Sources → Sync** does the same for that project's snapshot at its own style. Before v1.18.2 an unresolvable component blanked the **entire** preview with `X is not defined`; it is now isolated to a placeholder so the rest of the block still previews.
</Aside>

## Importing

<Tabs>
  <TabItem label="Admin UI">
    1. Go to **Admin → Block Builder → Import**.
    2. Paste your JSON or upload a `.json` file. Either format is detected automatically and a summary (component count, and field-template count for Builder exports) is shown.
    3. Click **Import**. The result reports how many components were **created** vs **updated**, plus any per-row errors.

    The **Export** button on the same tab downloads the current catalogue (plus `sharedFields`) in the Builder export format.
  </TabItem>
  <TabItem label="API — Builder format">
    ```bash
    curl -X POST https://your-app.example.com/api/v1/builder/components/import \
      -H "Content-Type: application/json" \
      -H "Cookie: <admin session>" \
      -d '{ "components": [ { "type": "button", "category": "element", "label": "Button", "jsxTag": "Button", "importPath": "@/components/ui/button" } ] }'
    ```
    Body: `{ "components": [ ... ] }`. Returns `{ imported, created, updated, errors }`.
  </TabItem>
  <TabItem label="API — shadcn format">
    ```bash
    curl -X POST https://your-app.example.com/api/v1/builder/components/import-json \
      -H "Content-Type: application/json" \
      -H "Cookie: <admin session>" \
      -d '{ "imports": { "ui-button": { "path": "@/components/ui/button", "components": ["Button"] } }, "components": { "button": { "category": "element", "label": "Button", "jsxTag": "Button", "importGroup": "ui-button" } } }'
    ```
    Body: `{ "imports": { ... }, "components": { ... } }`. Returns `{ imported, created, updated, errors }`.
  </TabItem>
</Tabs>
