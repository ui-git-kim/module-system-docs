The **Cog Ingest** module provides an admin-configurable content ingestion system powered by the [cognitive-db](https://github.com/ui-git-kim/cognitive-db) intelligence engine. You install one module (`npx cognitive-ingest init`) and get the full cognitive stack — perception, entity extraction, connection discovery, and ontology management.

Admins configure **ingest sources** (e.g. "Court Document", "Research Paper") with processing levels and extraction rules. Users submit content through the ingest UI — text, URLs, images, or PDFs — and the cognitive-db engine extracts entities, discovers connections, raises questions, and grows the knowledge graph. Results are stored as jobs with full history and stats.

<Aside type="tip" title="One module, full stack">
cognitive-db is bundled as a dependency, not installed separately. The module's schema includes all engine tables (`cog_node`, `cog_working_memory_event`, `cog_processing_log`) alongside the module tables. You configure the LLM and embedding providers during init — that's it.
</Aside>

## Quick Install

```bash
npx cognitive-ingest init
```

The CLI walks you through provider selection (LLM, embeddings), creates database tables, generates frontend and backend code, and installs shadcn components.

### Full Feature List

- Content ingestion powered by cognitive-db intelligence engine
- Four content types: text, URL, image (vision model), PDF
- Admin-configurable ingest sources with processing levels (minimal, standard, deep)
- Per-source configuration — context fields, extraction focus, prompt overrides, auto-tags
- Dynamic context fields on the ingest form driven by source configuration
- Entity extraction, connection discovery, and ontology growth
- Structure module integration — tabs and widgets on node detail pages
- Source-to-node-type mapping with custom extraction rules
- Ingest job history with detailed processing results
- Dashboard widget showing processing activity
- User preferences for default processing behaviour
- Settings page for ingestion configuration
- Batch ingestion support
- Extensible preprocessor registry for content transformation
- Filter and action hooks for pipeline customisation

## Architecture

```
┌─────────────────────────────────────────────────┐
│                  Frontend                        │
│                                                  │
│  Ingest Page ──► API Service ──► Backend Routes  │
│  Admin Page       ▲                   │          │
│  Dashboard Widget │                   ▼          │
│  Node Tab/Widget  │          Ingest Service       │
│                   │            │         │        │
│                   │    Preprocessors    Filters   │
│                   │            │                  │
│                   │            ▼                  │
│                   │      cognitive-db             │
│                   │       (bundled)               │
│                   │            │                  │
│                   │            ▼                  │
│                   │     LLM + Embeddings          │
│                   │     (Anthropic/OpenAI/Voyage) │
│                   │            │                  │
│                   │            ▼                  │
│                   │    Prisma (cog_* tables)      │
│                   └────────────┘                  │
└─────────────────────────────────────────────────┘
```

The ingest pipeline flows: **Content → Preprocessors → Filters (before) → cognitive-db → Filters (after) → Job Record**. Other modules can hook into any stage via the filter and action registries.

## Prerequisites

| Requirement | Details |
|-------------|---------|
| **Structure module** | Must be installed first (`requiredModules: ['structure']`) |
| **LLM provider** | Anthropic (Claude) or OpenAI (GPT) — API key required |
| **Embedding provider** | Voyage AI or OpenAI — API key required |
| **PostgreSQL with pgvector** | Required for semantic search embeddings |

## What Gets Installed

When you run `npx cognitive-ingest init`, the module generates:

**Frontend** (in `frontend/src/features/cog-ingest/`):
- 4 pages — main ingest, job detail, settings, admin
- 5 components — dashboard widget, node tab, node widget, preferences, source config dialog
- 3 hook files — ingest operations, source CRUD, job queries
- 1 service file — API client
- 7 registration files — navigation, dashboard, settings, admin, preferences, structure tab, structure widget

**Backend** (in `backend/src/features/cog-ingest/`):
- Routes, service, types, processors, config
- 6 registration files — route, service, action, filter, preprocessors, context-schema

**Database** — 6 Prisma models (3 engine + 3 module)

See [Configuration](/modules/cog-ingest/configuration/) for environment variables and init prompts, or [Usage](/modules/cog-ingest/usage/) for a walkthrough of the features.
