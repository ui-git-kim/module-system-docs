This page documents notable changes to the Cog Ingest module. The module uses semantic versioning: **major.minor.patch**.

- **Major** (1.0.0 → 2.0.0): Breaking changes requiring migration
- **Minor** (1.0.0 → 1.1.0): New features, backwards compatible
- **Patch** (1.0.0 → 1.0.1): Bug fixes, template fixes

<Aside type="tip">
Check your installed version with `npx cognitive-ingest info`.
</Aside>

---

## v1.3.6 — 2026-05-11

**Migrate settings page to current settings-blocks API**

The settings page template used deprecated component names from an earlier version of the starter template's `settings-blocks` package. This caused a runtime error (`SettingsDivider` not found) when navigating to the settings page.

- Replaced `SettingsSectionHeader` → `SectionHeader`
- Replaced `SettingsDivider` → `Divider`
- Replaced `SettingsDropdownRow` → `Row` + `Dropdown` (composition pattern)
- Replaced `SettingsToggleRow` → `Row` + `ToggleSwitch` (composition pattern)

---

## v1.3.5 — 2026-05-11

**Align templates with semantic token and shadcn conventions**

Brought all frontend templates into compliance with the current LLM rules and starter template conventions:

- Removed prohibited typography scale classes (`text-2xl`, `text-sm`, `text-xs`, etc.) from all page and component templates
- Removed prohibited font weight classes (`font-bold`, `font-semibold`, `font-medium`) from all templates
- Replaced raw HTML `<input>`, `<textarea>`, and `<select>` elements with shadcn `Input`, `Textarea`, and `Select` components
- Replaced `text-green-600` colour literal with `text-primary` semantic token
- Removed unused `Separator` imports from page templates
- Added `input` and `textarea` to `shadcnComponents` in module.ts; removed unused `switch`
- Removed undocumented `version` field from service registration metadata

## v1.3.3 and Earlier

Initial development releases. Core features:

- Content ingestion powered by cognitive-db intelligence engine
- Four content types: text, URL, image (vision model), PDF
- Admin-configurable ingest sources with processing levels
- Entity extraction, connection discovery, and ontology management
- Structure module integration (tabs and widgets on node pages)
- Ingest job history with detailed processing results
- Dashboard widget showing processing activity
- Filter and action hooks for pipeline customisation
- Batch ingestion support
- Extensible preprocessor registry
- User preferences and settings page
