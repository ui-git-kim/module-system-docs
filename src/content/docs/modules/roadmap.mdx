## The Vision

The Module System is WordPress for React SaaS â€” a registry-based architecture where self-contained modules can be installed, updated, and eventually managed directly from an admin dashboard.

The system builds **single-tenant SaaS applications designed for individuals and solo/small businesses**. These are focused, purpose-built tools â€” not enterprise platforms with complex org hierarchies and multi-tenancy. Think personal productivity apps, client-facing tools for freelancers, niche vertical SaaS for small business owners.

**Who builds with the system** is a separate question with two paths:

- **Developers** use the CLI workflow (`npx module-name init`) and the [Module Starter Template](/module-starter/) to create, customise, and publish modules.
- **Non-technical admins** will eventually be able to discover, install, configure, and update modules directly from the admin dashboard â€” just as WordPress admins browse and install plugins without touching a terminal.

The CLI is the developer path; the admin dashboard is the non-technical path to the same result.

<Aside type="caution">
Admin-installable modules require the module system to handle installation without CLI access. This means the init process (file generation, schema migration, registration) needs to run server-side. This is a significant architectural milestone that builds on top of the existing CLI workflow.
</Aside>
