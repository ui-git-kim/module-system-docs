## Frontend Routes

| Path | Component | Description |
|------|-----------|-------------|
| `/:nodeType` | `NodeListPage` | List view with filtering (e.g., `/clients`) |
| `/:nodeType/:nodeId` | `NodeDetailPage` | Detail page with default tab |
| `/:nodeType/:nodeId/:tabId` | `NodeDetailPage` | Direct tab deep link |

<Aside type="note">
Routes use catch-all patterns and are inserted at the end of the route table so that more specific routes from other modules take precedence.
</Aside>

## Detail Pages

When a user navigates to a node (e.g., `/clients/abc123`), they see a detail page with configurable layout.

### Page Structure

1. **Header** (`NodeDetailHeader`) - Node icon, name, favourite toggle, and dropdown menu with actions (edit, archive, delete)
2. **Breadcrumbs** - Dynamic breadcrumbs in the site header showing `Type (plural) > Node Name`
3. **Navigation** - Style depends on node type configuration
4. **Tab Content** - The selected tab's content

### Navigation Styles

Admins configure the navigation style per node type in the [Pages tab](/modules/structure/admin/#pages-tab):

| Style | Layout | Best For |
|-------|--------|----------|
| **Tabs** | Horizontal tab bar above content | Few tabs, quick switching |
| **Sidebar** | Left navigation panel with right content area | Many tabs, complex layouts |
| **None** | Single content area, no navigation | Simple nodes with one view |

### Breadcrumbs

The Structure module uses the starter template's dynamic breadcrumb system:

- **List page** (`/clients`): Shows `Clients` in the header breadcrumb
- **Detail page** (`/clients/abc123`): Shows `Clients > Acme Corp` where "Clients" links back to the list

This uses the `useBreadcrumbs()` hook from `@/components/layout` to set breadcrumbs dynamically after the node data loads.

### Built-in Tabs

Structure provides these default tabs (all can be hidden per node type):

| Tab | Description |
|-----|-------------|
| **Overview** | Dashboard-style widgets from various modules |
| **Info** | Custom fields defined for this node type |
| **Children** | Child nodes with create/reorder support |
| **Connections** | Related nodes via the connection system (create, view, delete) |
| **Settings** | Node-specific settings |

### Connections Tab

The Connections tab provides a full CRUD interface for managing relationships between nodes:

- **View connections** - See all nodes connected to the current node with their type and connection label
- **Create connections** - Use the `NodeSelector` dialog to pick a target node and specify a connection type
- **Delete connections** - Remove connections with confirmation

Connections are stored in the `structure_node` table using `fromNodeId`, `toNodeId`, and `connectionType` fields.

### Adding Tabs from Other Modules

Other modules register tabs via the **Node Tab Registry**:

```typescript
import { nodeTabRegistry } from '@/features/structure';

nodeTabRegistry.register({
  id: 'tasks',
  moduleId: 'tasks',
  label: 'Tasks',
  icon: 'CheckSquare',
  nodeTypes: '*',
  order: 20,
  component: TasksTab,
  isVisible: (node, nodeType) => {
    return nodeType.features.tasks !== false;
  },
});
```

The tab automatically appears on node detail pages for matching node types. See [Registries](/modules/structure/registries/) for the full registry API.

## List Pages

Each node type gets a list page at `/:nodeType` (e.g., `/projects`).

### Views

Three display modes, configurable per node type:

| View | Description |
|------|-------------|
| **Grid** | Card-based layout with configurable card content |
| **List** | Compact row-based layout |
| **Table** | Column-based table (uses configured columns) |

### Filtering

When a node type has `filterableFields` configured, the list page renders filter dropdowns automatically. Users can filter nodes by select/multi-select field values.

Filters work with the search bar - both are combined to narrow results.

### Card Configuration

How cards display in grid/list views is controlled by the `card` configuration in the node type's `listPage` settings. Admins configure this visually in the [Admin Pages tab](/modules/structure/admin/#pages-tab) with the live card preview.
