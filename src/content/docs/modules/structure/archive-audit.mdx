A running log of **archive-readiness audits**. Before a working session on the Structure module is archived, it is audited for anything that would be lost — unfinished work, loose ends, decisions, and ideas — and the findings are recorded here so a fresh session can pick them straight up.

<Aside type="note" title="Why this page exists">
Session transcripts get archived and are then awkward to search. This page is the durable, reviewable home for "what was in that session" — decisions and their reasoning, ideas raised in passing, work left mid-flight, and anything the code and the rest of these docs don't already capture. Newest entry first. Nothing here is deleted; when an item is actually done, it's ticked and left in place for provenance.
</Aside>

## How to use this page

At the end of a session you want to archive, run the archive-readiness prompt (below). It audits the session and **appends a new entry at the top** of the log, using the entry template. Then it builds and pushes the docs. Review the entry on the live site; archive the session; start the next one by reading the open items here.

### The prompt

> **Archive-readiness check.** Audit this session end-to-end for anything that would be lost if I archive it: (1) unfinished work — started-not-finished, committed-not-pushed, coded-not-validated, or left WIP (check `git status` and recent commits in every repo touched, including the `docs/` submodule — **and if this is an older session the repo has since moved past, expect HEAD, the version number, and the working tree to look _newer_ than this session's output, and its commits to sit deep in history, absent from a short `git log`/reflog window; that is normal forward progress, not loss. Confirm the work survived by identifying this session's own commits from the transcript and checking each is an ancestor of `origin/main` (`git merge-base --is-ancestor <sha> origin/main`) — reachable from origin means preserved even though HEAD is far ahead. Real loss is only a commit that is on no branch _and_ absent from `origin`; do not infer loss from "HEAD isn't at my version" or "the working tree differs from what I edited"**); (2) loose ends — TODOs, "later" items, deferred fixes, promised follow-ups, checks I said I'd run; (3) decisions & ideas — any decision, trade-off, rejected approach and *why*, or idea raised in conversation not already written in code or docs; (4) undocumented reality — anything we changed or learned the docs don't reflect. Then **append a new entry to the top of the Archive Audit log** (`docs/src/content/docs/modules/structure/archive-audit.mdx`) using the entry template already on that page. Build the docs, push the docs submodule **and** the submodule pointer. Verify before ticking anything "done" (if a version was bumped, confirm `dist` was rebuilt after the bump; for an older session, confirm `dist` — if gitignored — stamps the **current HEAD** version, not the session's). Finish by telling me plainly whether it's safe to archive.

### Entry template

```mdx
## YYYY-MM-DD — <session focus>

**Verdict:** ✅ Safe to archive  ·  ⚠️ Needs input first  ·  ℹ️ Superseded (work integrated, session now historical)
**Shipped this session:** <module version(s), key commits, what went live>

**Unfinished / WIP:** <or "none">
**Loose ends & follow-ups:** <deferred fixes, promised checks, TODOs — or "none">
**Decisions & ideas:** <decisions + why, rejected approaches, ideas raised — or "none">
**Recorded where:** <docs pages / memory files updated, so it's findable>
**Open items for next session:** <the pick-up list — or "none">
```

A cheap code-vs-docs reconcile sweep (grep every `doActionAll(` call site + every doc hook row, diff by name and payload keys) catches the whole class at once; it found the `duplicated` and `workflowStateChanged` drifts and the `nodeTypeGroup` omissions in one pass.
