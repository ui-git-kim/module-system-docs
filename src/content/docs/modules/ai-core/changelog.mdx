This page documents notable changes to the AI Core module. The module uses semantic versioning: **major.minor.patch**.


- **Major** (1.0.0 → 2.0.0): Breaking changes requiring migration
- **Minor** (1.0.0 → 1.1.0): New features, backwards compatible
- **Patch** (1.0.0 → 1.0.1): Bug fixes, template fixes

<Aside type="caution" title="Design stage — not yet built">
The AI Core redesign is design-complete (July 2026). Implementation has not started; the currently installable module ships an earlier, superseded surface. See the [Roadmap](/modules/ai-core/roadmap/) for status.
</Aside>

<Aside type="tip">
Check your installed version with `npx ai-core info`.
</Aside>

## Unreleased

**July 2026 alignment — infrastructure only, no version bump**

An infrastructure alignment carried out in July 2026, alongside publication of the redesign. Per the versioning policy, infrastructure-only changes do not bump the module version; this entry records them for the audit trail.

- **Scaffold adopted at v1.19.1** (module contract v1.2.0): lifecycle hooks, the migrations runner, uninstall/enable/disable support, the contract npm scripts, and the docs submodule
- **Module version reconciled to the 1.x line.** The version had been advanced to an abandoned, never-shipped 2.0.0; it has been brought back to 1.x, matching the redesign's stated version line — nothing downstream imports the current surface, so it can be replaced cleanly
- **CLAUDE.md adopted the SCAFFOLD marker contract**

**An honest record of the abandoned "v2".** Between the January release and the redesign, a second surface was built in-repo — AI SDK 6 transport, purpose keys, image and video generation, embeddings, model discovery — but it was **never released and never installed anywhere**. It is superseded by the published redesign and will be replaced during implementation. The redesign's shape — models as records, operations as the public surface, disciplines and guardrails — is documented on the [overview](/modules/ai-core/) page, and the path from here to a shippable release is on the [Roadmap](/modules/ai-core/roadmap/). Scaffold and contract mechanics are covered in the [developer workflow](/modules/ai-core/developer-workflow/).

## v1.0.0 — 2026-01-23

**Initial release**

The first installable AI Core module — shared AI infrastructure consumed by other AI-powered modules.

- **Providers** — SDK initialisation for Anthropic and OpenAI, lazily initialised on first use
- **Model capability discovery** — dynamic capability fetching with caching and conservative fallbacks when the provider API is unavailable
- **Content processors** — image (vision), URL content extraction, and PDF processing
- **Prompt registry and assembler** — a composable prompt system; consumers register their own prompts and assembly injects date context automatically
- **Utilities** — token estimation and typed AI error handling

This was the released surface. The repo's templates were since overwritten by the never-released v2 work (see the [Roadmap](/modules/ai-core/roadmap/) Known Issues); the published redesign supersedes both.
