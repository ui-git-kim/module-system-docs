## Known Issues

Issues discovered but not yet fixed. All of these came out of the July 2026 alignment audit, and all of them live in the **legacy template surface** — the never-shipped v2 code that the redesign replaces wholesale. They are recorded here so nobody rediscovers them; none of them will be fixed in place.

### Legacy Template Surface Is Superseded

**Severity:** High (for anyone tempted to install)
**Where:** `src/templates/backend/index.ts`, `src/templates/frontend/index.ts`

The templates in this repo ship the never-shipped v2 surface — purpose keys, content processors, image/video generation. The redesign replaces that surface wholesale: models become admin-managed records, consumers call operations by name (see [Usage](/modules/ai-core/usage/)), and content acquisition moves out of AI Core entirely (see [Integration](/modules/ai-core/integration/)). Do not build on the current templates.

**Workaround:** Wait for the implementation, or pin expectations to the design pages ([Overview](/modules/ai-core/), [Usage](/modules/ai-core/usage/), [Configuration](/modules/ai-core/configuration/)).

### Generated `processors/types.ts` Does Not Compile

**Severity:** Low (superseded)
**Where:** `src/templates/backend/index.ts` — lines ~946, ~978, ~983

Triple-escaped template literals in the processor type templates emit invalid TypeScript, so the generated `processors/types.ts` does not compile. Superseded by the redesign — processors are not part of the new surface, so the file is replaced rather than repaired.

**Workaround:** None needed — do not build on the legacy surface.

### `getModel` String Overload Untypeable, Duplicate `ModelPurpose` Barrel Export

**Severity:** Low (superseded)
**Where:** `MODEL_REGISTRY` template and the backend `INDEX_TEMPLATE` in `src/templates/backend/index.ts`

The `getModel` string overload cannot be expressed in the type system, and `ModelPurpose` is exported twice from the generated barrel. Superseded by the redesign — purpose keys disappear entirely. A consumer names a job, never a model, and resolution happens in the operations layer (see [Usage](/modules/ai-core/usage/)).

**Workaround:** None needed — do not build on the legacy surface.

### AI SDK 6 API Mismatches

**Severity:** Low (superseded)
**Where:** `src/templates/backend/index.ts` — provider and generation templates

The templates call `LanguageModelV1`, `generateImage`, and `voyageai` package APIs that do not exist under the pinned dependencies, so the generated provider layer cannot build. Superseded by the redesign, which targets Vercel AI SDK v7 behind an owned provider interface — the SDK appears in one adapter layer and nowhere else (see [Integration](/modules/ai-core/integration/)).

**Workaround:** None needed — do not build on the legacy surface.

### Six Install Prompts Emit Literal `{{placeholders}}`

**Severity:** Low (superseded)
**Where:** `src/module.ts` — install prompt configuration

Six install-time values — `visionModel`, `analysisModel`, `autonomousModel`, `researchModel`, `predictiveModel`, `videoGenProvider` — have no prompt and no default in `module.ts`, so the generated config emits literal `{{placeholders}}`. Superseded by the redesign, where model assignments are records managed in the admin rather than install-time config (see [Configuration](/modules/ai-core/configuration/)).

**Workaround:** None needed — do not build on the legacy surface.

### Backend Templates Use `console.*` and Never Use Their Own `AIError` Helpers

**Severity:** Low (superseded)
**Where:** `src/templates/backend/index.ts` — service and processor templates

The backend templates log with `console.*` — banned by the LLM Rules, which require `createModuleLogger` — and never call the `AIError` helpers they themselves export. Superseded along with the rest of the legacy surface; the implementation follows the module conventions from the start (see [Developer Workflow](/modules/ai-core/developer-workflow/)).

**Workaround:** None needed — do not build on the legacy surface.

## Planned Improvements

### Short-Term — The Implementation

The short-term column is the redesign itself, briefed from these pages. Each item below is a coherent slice a development session can pick up; the [Developer Workflow](/modules/ai-core/developer-workflow/) page describes how those sessions run.

#### `ai_`-Prefixed Schema via the Migrations Runner

The full table set, delivered through the per-module migrations runner (migrations, never `prisma db push`): models, operations, prompt segments, disciplines, guardrail checks, flows plus persisted flow state, the usage ledger, and the review queue. Every built-in the module ships is a row in one of these tables — seeded at install, editable, resettable. **Ships-with is not hardcoded.**

#### The Operations Resolver and `aiCore.run` Consumer API

The whole public surface:

```ts
const result = await aiCore.run('extract-entities', input);
```

A consumer names a job, never a model. Resolution selects the model record, applies its parameter rules, assembles the prompt through the model's framework, and returns the result with structured feedback. The acceptance test from the design: **swapping one model for another across an entire app is one admin change, and nothing else moves** — no file edits, no redeploy, no consumer aware.

#### Provider Adapter Layer on Vercel AI SDK v7

The SDK is transport, not architecture: it appears in the provider adapters only, satisfying an interface AI Core defines, and uses only the generation, embedding and rerank primitives. Parameter validation sits above the SDK, against the model record's own rules — a silently clamped parameter is a lie about what ran, so AI Core rejects or transforms explicitly, visibly, and in the ledger. The exit test: swapping the SDK touches the adapter layer and nothing else.

#### `security.registration.ts`

Deletion handlers, encrypted fields, and the security manifest. AI Core has user-linked data (the ledger, flow state, the review queue) and must not be the gap in the app's row-level security. The design's phrasing of the obligation: *don't be the new hole.*

#### Admin Page

One page, tabbed — Models, Operations, Prompts, Dials and presets, Disciplines, Flows, Usage — registered via `adminPagesRegistry` at order 30 or later. Cost appears in help text, not just numbers, and every value shows where it came from: a shipped default or someone's deliberate choice. See [Configuration](/modules/ai-core/configuration/).

#### Flows with Suspend/Resume

An operation may wait on the previous operation's feedback, and that is correct — analysing behaviour over messages whose senders could not be identified produces confident output over unresolved data. The flow suspends and resumes, which costs persisted flow state and a resumption path. Quality over speed; build it.

#### Seeded Batteries

The generic voices, the generic disciplines, and the framing segments that introduce external material — all authored records, seeded at install like a theme's default palette. You can change every colour; you just don't start from white.

### Medium-Term

#### Groundedness Scorers

A pluggable guardrail type, configured per operation: provider-native citations, claim decomposition plus entailment, or a dedicated grounding check. The score is what the abstention threshold compares against, and it is an input to the feedback an operation returns — never a certificate that the output is true. It belongs on retrieval-heavy operations, not on everything.

#### Semantic Cache

AI Core sees all provider traffic, so every consumer benefits without knowing the cache exists. Cache keys are scoped to the owner, hard — a cache keyed on similarity alone will serve one person's answer to another, and that is a real, reproduced leak. Sensitive content is not cached, or cached only where provider terms permit.

#### Re-embedding Migration

AI Core owns the embedding model, so it owns the moment the model changes: the existing vectors are invalid, and only AI Core knows it. Report the scale, then migrate in batches with progress. Silently leaving stale vectors in place is the same failure as silent truncation.

#### Review-Queue Battery

A table and a component a consumer can use or ignore when feedback has nobody to ask right now — not a mechanism, not a destination, not routing. Just something in the box.

#### Usage Ledger Surfaced in Admin

What ran, which model, tokens in and out, latency, outcome — enough to answer *"which model actually does this job well"* with data instead of an opinion, and enough to make a past decision explicable: exact model version, prompt and discipline versions, retrieval set as IDs, parameters actually sent, provider terms in force. All of it by reference — IDs, versions and hashes, never content.

### Long-Term / Open Questions

The design's own open questions (§21), carried here so they stay visible.

#### Discipline Authority Enforcement at the Startup Barrier

The design's original question — can a registry reject a registration that conflicts with another registry's contents? — was answered in July 2026: registration-time rejection is unsound in the starter's ESM side-effect model. What remains open is the implementation: a hard-failing referential-integrity pass at the backend startup barrier, the CI lint for orphaned checks and unreferenced rules, and docs that state plainly that the hierarchy is advisory during registration. The design's honesty rule applies to the module's claims about itself: do not quietly ship a hierarchy that doesn't hold. The principle underneath: **prose asks; code holds.**

#### Which Groundedness Scorer per Operation

The mechanism is settled and pluggable; which implementation runs on which operation is a per-operation decision that needs measurement, not a design argument.

#### Image and Video Provider Data-Handling Terms

Black Forest Labs and Replicate have materially weaker data-handling terms than the model providers. Confirm before any sensitive content is routed to them — the likely answer is that it never is, enforced by the sensitivity gate in resolution.

#### Chat Module or Generic Chat

If chat goes generic, persona has nowhere natural to live. Persona is the user's, conversation-scoped, and only chat has conversations — chat owns the records and hands AI Core a segment.

#### Naming Collisions

`prompt`, `token` and `model` collide with the module system's own terms. A naming pass is needed before the admin surface freezes.
