## Standard Layout

<FileTree>
- module-name/
  - frontend/src/features/[module]/
    - index.ts
    - navigation.registration.ts
    - routes.config.tsx
    - components/
    - pages/
    - registrations/
  - backend/src/features/[module]/
    - index.ts
    - [module].routes.ts
    - [module].service.ts
    - route.registration.ts
    - service.registration.ts
  - prisma/
    - schema.additions.prisma
  - docs/
    - README.md
    - INSTALLATION.md
</FileTree>

---

## Naming Conventions

| Type | Convention | Example |
|------|------------|---------|
| Module folder | lowercase | `billing` |
| Routes | `[module]Routes` | `billingRoutes` |
| Service | `[module]Service` | `billingService` |
| Hook | `use[Module]` | `useBilling` |

---

## Index File Pattern

```typescript
// frontend/index.ts
import './navigation.registration';
import './registrations/dashboard.registration';

export { moduleRoutes } from './routes.config';
export { useModule } from './hooks/useModule';
```

```typescript
// backend/index.ts
import './route.registration.js';
import './service.registration.js';

export { moduleRoutes } from './[module].routes.js';
export { moduleService } from './[module].service.js';
```

## Order Convention

| Range | Purpose |
|-------|---------|
| 1-9 | Core/system |
| 50-79 | Standard modules |
| 100+ | Plugins |

## Documentation Requirements

Every module needs:
- `README.md` - Overview and features
- `INSTALLATION.md` - Step-by-step setup
- `CHANGELOG.md` - Version history

---

## Publishing Checklist

- [ ] Follows naming conventions
- [ ] Uses registries only
- [ ] TypeScript compiles
- [ ] Tests pass
- [ ] Documentation complete
- [ ] Installation tested
