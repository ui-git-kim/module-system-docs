All notable changes to **cognitive-db** are recorded here. The engine uses semantic versioning (`major.minor.patch`) and follows the [module system's versioning conventions](/getting-started/). It ships vendored inside the [Cog Ingest module](/modules/cog-ingest/), so an engine release reaches apps through the next module release.

  This page mirrors the engine repository's `CHANGELOG.md`. For where the project is heading, see the [Roadmap](/cognitive-db/roadmap/); for what the engine is and does, see the [Overview](/cognitive-db/).
</Aside>

## 1.0.0 — 2026-07-14

First stable release. Hardened by a comprehensive code audit and moved to the module system's vendored delivery. This marks a stable public API — semantic versioning applies from here.

### Fixed — audit remediation

84 findings across 96 files (see `docs/AUDIT-2026-07-14.md` in the engine repository).

| Area | Remediation |
|------|-------------|
| **Security** | SQL injection in graph search closed — every filter value now binds as a numbered placeholder (`QueryParams`); scaffold `execSync` no longer interpolates the target path; `retentionDays` bound in cleanup SQL. |
| **Correctness** | Graph adjacency now resolves the far endpoint (centrality / community / pathfinding had been operating on connection ids); atomic `createVersion` (`$transaction`); `userId` set on relationship connections; connection-limit truncation removed; version-chain cycle guards; preference-detection valence parsing; and roughly 40 more. |
| **Performance** | DB-side aggregation instead of full scans; a single batched connection fetch; windowed trend queries. |

<Aside type="caution" title="Security model">
  The SQL-injection and binding fixes are query-hygiene hardening. They are distinct from the engine's isolation model, which is **ownership by `userId`** (`tenantId === userId`, the solo-business model) — there is no multi-tenant layer. See [Security & data ownership](/cognitive-db/) for the full picture.
</Aside>

### Changed — minor API

- Removed the unused `semanticWeight` option from `HybridSearchOptions` — it was never applied, because `hybridSearch` is **filtered semantic search** (metadata-filtered vector search, not a lexical blend).
- `matchEntities` now keys its result map by input index instead of `name:type`, avoiding collisions.
- Stricter types on several exported functions (proper unions, explicit return types).
- Extended the minimal `PrismaClientInterface`; added `NodeStatus.ANSWERED`, `FindNodesOptions.createdAfter` / `createdBefore`, and `getConnectionsForNodes()`.

### Added

- A `prepare` script builds `dist/` on install, enabling Git and vendored installs without a published package.
- `core/node.mapper.ts` — a single shared row-to-`Node` mapper, deduplicated from four call sites.

### Delivery

- Vendored into the Cog Ingest module and installed via its `onInstall` / `onUpdate` lifecycle hook — no npm registry or GitHub authentication required, and the engine version is locked to the module release. A standalone install is also available via `npm install github:ui-git-kim/cognitive-db`.

### Design

- Confirmed **`tenantId === userId`** as the tenant model (solo-business focus): isolation is by `userId` ownership, with no multi-tenant layer.
